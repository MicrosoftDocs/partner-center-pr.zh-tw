## <a name="updated-promotions-for-office-365"></a><a id="14"/>已更新 Office 365 的促銷</a>

### <a name="categories"></a>類別

- 日期：2020-04-30
- 輸入：功能



### <a name="summary"></a>摘要

Microsoft 將於 2020 年 5月4日 (UTC 上午 12:01) 推出 Office 365 E1 與 Microsoft 365 商務基本版的新促銷，以取代目前提供給新客戶的 Office 365 E1 6 個月試用。

### <a name="impacted-audience"></a>影響對象

透過雲端解決方案提供者 (CSP) 方案交易的所有合作夥伴。 此促銷僅適用於全新的商業客戶。

### <a name="details"></a>詳細資料

在這段空前嚴峻的時期，[Microsoft 向客戶承諾](https://www.microsoft.com/microsoft-365/blog/2020/03/05/our-commitment-to-customers-during-covid-19/)，在雲端解決方案提供者中為新客戶 (請參閱下方定義) 提供 Office 365 E1 6 個月試用。
Microsoft 將於 2020 年 5 月 4 日推出 Office 365 E1 與 Microsoft 365 商務基本版的新折扣促銷，透過按月計費的年度訂閱為新客戶供應這些產品，前 6 個月免費。

這些促銷將取代目前的 Office 365 E1 試用供應項目，相異之處如下所示：

- 合作夥伴將利用前 6 個月享有 100% 折扣的雲端解決方案提供者促銷 (而不是唯一的 6 個月試用 SKU)，來交易年度 Office 365 E1 或 Microsoft 365 商務基本版 SKU。 [深入了解雲端解決方案提供者促銷](../promotions.md)。
- 我們會從技術上限制「只有新客戶」能夠進行訂購，而不是像 Office 365 E1 試用，僅透過書面原則限制只有新客戶可以使用。 
- 對於 SMB 客戶，Microsoft 365 商務基本版將包含在此促銷活動中。 這可確保合作夥伴能使用 Microsoft 365 商務版，讓客戶在正確的產品系列中從少於 300 個基座開始。
目前已獲授權使用 Office 365 E1 試用的客戶 (可於 2020 年 5 月 4 日之前取得) 將保留目前使用的方案，這項變更只會影響全新客戶。

新客戶的定義

有資格享有此促銷的新客戶為：

1. 全新客戶租用戶
2. 現有客戶的租用戶中，對於 Microsoft 合作夥伴常見問題集 (FAQ) 指南中定義的任何產品，沒有目前已付費或最近取消 (過去 30 天內) 的訂閱

 
### <a name="additional-resources"></a>其他資源

- 若要深入了解安全遠距工作，請前往[合作夥伴遠距工作資源中心](https://www.microsoft.com/microsoft-365/partners/remotework)。


### <a name="next-steps"></a>接下來的步驟

請參閱 2020 年 5 月的價目表，以了解上述變更，並與組織中所有相關連絡人分享這項資訊。

## <a name="azure-reservations-new-customer-self-serve-capability-with-partner-permission"></a><a id="13"/></a>具有合作夥伴權限的 Azure 保留新客戶自助服務功能

### <a name="categories"></a>類別

- 日期：2020-04-29
- 輸入：功能
- 狀態：現已推出
- [深入了解](https://partner.microsoft.com/resources/collection/customer-permission-to-purchase#/)

### <a name="impacted-audience"></a>影響對象

雲端解決方案提供者計畫中的合作夥伴

### <a name="summary"></a>摘要

Microsoft 致力於提供彈性的選項，讓客戶更容易向 Microsoft 和我們的合作夥伴購買及使用產品與服務。

一直以來，我們承諾要為 Azure 保留，推出新的客戶自助服務功能，以簡化客戶的購買體驗而增加透過 CSP 來使用 Azure 保留的機會。

Azure 保留是以折扣價購買 Azure 服務的絕佳方式。 藉由這項新功能，合作夥伴可以選擇授權客戶，在自己為客戶購買的既有 Azure 訂用帳戶中，購買自己的資源。

以下詳述此功能的運作方式：

- 合作夥伴可以選擇在個別客戶帳戶層級啟用此功能，將購買權授與客戶。
- 根據預設，此功能會設為 [關閉]，合作夥伴可選擇啟用並授權給客戶。
- 合作夥伴將能在合作夥伴中心內驗證及管理客戶的購買事宜。
- API 可用於大規模啟用客戶的權限，以自動執行此流程。

### <a name="next-steps"></a>接下來的步驟

1. 使用這項新功能，開始協助您的客戶購買自己的 Azure 保留。
2. [檢閱此集合所提供的資源](https://partner.microsoft.com/resources/collection/customer-permission-to-purchase#/)，其所含的詳細資訊包括[技術文件](../give-customers-permission.md)、合作夥伴體驗逐步解說及客戶體驗逐步解說。

## <a name="microsoft-reversed-the-csp-eur-price-for-microsoft-365-e3"></a><a id="12"/></a>Microsoft 回復了 Microsoft 365 E3 CSP 的歐元價格。

### <a name="categories"></a>類別

- 日期：2020-04-24
- 輸入：供應項目/市場
- 狀態：現已推出
- [深入了解](https://flow.microsoft.com/ui-flows/)

### <a name="summary"></a>摘要

Microsoft 於 2020 年 4 月 1 日修正了 Microsoft 365 (M365) E3 的歐元定價錯誤。 但在 2020 年 4 月 22 日，Microsoft 暫時回復了價格，協助合作夥伴應對這些前所未有的情況。

### <a name="impacted-audience"></a>影響對象

所有參加雲端解決方案提供者計畫，並使用歐元價格表，進行 M365 E3 交易的合作夥伴。

### <a name="details"></a>詳細資料

我們在 2020 年初，發現 CSP 的 Microsoft 365 E3 歐元價格表有誤，網路價格低於應有的價格。 我們排程了系統更新，於 2020 年 4 月1 日更正價格，修訂了該供應目的價格點。

面對這些前所未有的情況，儘管定價有誤，但目前仍不是修正及提高價格的時機。 因此，我們暫時回復到先前比較低的網路定價，並自 2020 年 4 月 22 日起生效。 所有發票都會立即反映先前較低的定價，而且很快地自 2020 年 5 月起，價格表也會顯示較低的定價。

請合作夥伴不要使用 2020 年 4 月價格表上所列的 M365 E3 歐元定價，而請改用 2020 年 3 月價格表所列的定價。

我們非常感謝合作夥伴社群，在面對此前所未有的情況下所做的一切。

### <a name="next-steps"></a>接下來的步驟

請與貴組織所有相關連絡人分享這項資訊。

### <a name="questions"></a>有任何問題嗎？

在與 Microsoft 合作上如需任何支援，請連絡支援人員協助回答您的問題、查詢或更多資訊。

## <a name="reminder-activation-of-security-safeguards-for-partner-center-transactions-in-csp"></a><a id="11"/></a>提醒：為雲端解決方案提供者中的合作夥伴中心交易啟用安全性保護

### <a name="categories"></a>類別

- 日期：2020-04-23
- 輸入：功能
- 狀態：即將推出
- [深入了解](https://partner.microsoft.com/resources/collection/partner-security-requirements#/)

### <a name="target-partner-audiences"></a>目標合作夥伴對象

- 所有參與雲端解決方案提供者 (CSP) 計畫的合作夥伴
- 所有控制台廠商 (CPV)
- 所有 Advisor 合作夥伴

### <a name="summary"></a>摘要

更好且持續的安全性和隱私權保護是我們最優先的考量，而且我們一直在協助合作夥伴保護其客戶和租用戶。 在 2019 年，我們推出了[強制安全性需求](https://blogs.partner.microsoft.com/mpn/safeguard-business-security-best-practices/)。 所有參與雲端解決方案提供者計畫的合作夥伴、控制台廠商及 Advisor 都應該實作這些需求，以確保符合規範。

我們最近已順利完成為所有合作夥伴租用戶啟用代理管理 (Admin-On-Behalf-Of，AOBO) 功能的安全性保護，以保護其企業與客戶免於身分盜用的相關事件。

為了進一步協助保護合作夥伴與客戶，**自 2020 年 5 月 1 日起，我們將開始為雲端解決方案提供者中的合作夥伴中心交易啟用保護**。 啟用額外的安全性保護，表示合作夥伴必須使用多重要素驗證 (MFA)，來處理雲端解決方案提供者中透過合作夥伴中心使用者體驗或 API 的任何交易，以避免未經授權的存取，並保護其客戶與租用戶免於身分盜用的相關安全性事件。 如需詳細資訊，請參閱[此處](../partner-security-requirements-mandating-mfa.md)。

### <a name="next-steps"></a>接下來的步驟

- **如果您尚未啟用建議的安全性需求來保護您的租用戶及使用者**，請立即進行，以確保符合規範，以及避免管理客戶或在雲端解決方案提供者中交易時的任何業務中斷。 未實作安全性需求的合作夥伴，可能會使他們在參與雲端解決方案提供者計畫時面臨風險。
- 間接提供者必須在自己的公司實作這些安全性需求，並利用此電子郵件範本，確認轉銷商也實作了這些需求。 我們也會透過電子郵件，直接連絡您的轉銷商。
- 請前往[合作夥伴中心安全性需求狀態頁面](https://partner.microsoft.com/commerce/security/compliance)，以識別進度及需要採取的主要動作。 與您的全域管理員合作，定期檢查各個租用戶的安全性需求進度。
- 如果您使用的是控制台廠商 (CPV) 解決方案，請洽詢您的 CPV。

### <a name="additional-resources"></a>其他資源

請檢閱下列資源，以協助您實作安全性需求。

- 請查看[逐步指南](../partner-security-requirements.md)以實作合作夥伴安全性需求。
- 請參閱[常見問題集](../partner-security-requirements-faq.md)，以及合作夥伴安全性需求相關問題的回答。
- 請檢視[安全性需求狀態檢查指南](../partner-security-compliance.md)。
- 請存取[實作合作夥伴安全性需求](https://partner.microsoft.com/resources/collection/partner-security-requirements#/) (英文) 集合的其他資源。
- 請將[合作夥伴中心安全性指導社群團體](https://aka.ms/MPCSecurityGuidance) (英文) 加入書籤。

如需解決技術問題的其他協助，請[提交支援票證](https://partner.microsoft.com/dashboard/support/csp/servicerequests/create?topicid=32674991)。 請將此電子郵件轉寄給組織中的其他人，這些人員可能需要採取動作來實作這些安全性需求。

感謝您的合作，也感謝您為保障生態系統的運行得以信賴，所付出的一切努力。

## <a name="announcing-microsoft-defender-advanced-threat-protection-mdatp-for-servers-offer-available-for-csp-on-april-1-2020"></a><a id="9"/></a>宣布於 2020 年 4 月 1 日為雲端解決方案提供者推出適用於伺服器的 Microsoft Defender 進階威脅防護 (MDATP) 供應項目。

### <a name="categories"></a>類別

- 日期：2020-04-17
- 輸入：供應項目/市場
- 狀態：現已推出

### <a name="summary"></a>摘要

我們將於 2020 年 4 月 1 日為雲端解決方案提供者適用於伺服器的 Microsoft Defender ATP SKU，價格為每月每個 OSE 美金 5 元。

### <a name="impacted-audience"></a>影響對象

這項新的 SKU 適用於想要使用 Microsoft Defender ATP 進行伺服器端點保護，並至少有 50 個授權 (MDATP、Win E5、M365 安全性、M365 E5) 的 Microsoft Defender 進階威脅防護 (ATP) 客戶。

### <a name="details"></a>詳細資料

Microsoft 針對 (內部部署和 Azure 中) 伺服器端點保護推出的供應項目一直是 Azure 資訊安全中心標準。 Azure 資訊安全中心標準費用為每月每個伺服器作業系統環境美金 15 元。 客戶向我們反映，希望有一個功能較少且價位較低的選項，能夠透過與 E5 相同的購買方式來取得伺服器端點保護。
為了回應客戶的需求，我們新推出適用於伺服器的 Microsoft Defender ATP SKU 來滿足客戶的需求，並預期使用此 SKU 的多數客戶都能夠為內部部署伺服器取得保護，但不限於該使用案例。 Azure 資訊安全中心標準仍然是具有一系列擴充威脅防護功能的主打 SKU，而且合作夥伴應利用 Azure 資訊安全中心標準主打 SKU 來引導與客戶的交談，特別是具有 Azure 工作負載的客戶。 不過，現在 Microsoft 可以提供價位較低的端點保護，讓 Microsoft 合作夥伴使用適用於伺服器的 Microsoft Defender ATP SKU，在端點保護市場中能夠更有競爭力。

至少有 50 個授權的 Microsoft Defender ATP 客戶可以使用全新適用於伺服器的 Microsoft Defender ATP SKU 進行伺服器端點保護，費用為每月每個伺服器作業系統環境美金 5 元。 若要增強伺服器的雲端工作負載保護，適用於伺服器的 Microsoft Defender ATP SKU 客戶每月可針對每個伺服器 OSE 額外支付美金 $10 元，升級為 Azure 資訊安全中心標準 SKU，並享有多項功能的權益，例如自適性應用程式控制、Just-In-Time 虛擬機器存取、檔案完整性監視，以及用於保護 Azure 資源和取得附加價值的整合體驗 (例如未來與 Azure Arc 整合)。

Azure 資訊安全標準目前沒有任何變更。 Azure 資訊安全中心標準會持續提供整合式使用者體驗，不僅保護伺服器虛擬機器端點，也保護多種 Azure 資源，包括 IoT、Azure Kubernetes Services (預覽)、Azure Container Registry (預覽)、Key Vault (預覽)、應用程式服務、SQL 和儲存體。

適用於伺服器的 MDATP SKU 擁有者必須透過美金 10 元的升級 SKU 升級為 Azure 資訊安全中心 (ASC) 標準，才能取得 ASC 標準功能的權益。

**歡迎您提供任何意見！** 當您使用新的作業整備位置時，歡迎您留下意見反應，讓我們可以持續改善您的體驗。 若要這麼做，您可以使用每個頁面底部所提供的 **此頁面有所助益嗎？是 | 否** 意見反應工具。

Microsoft 將會簡化通知方式，並提供所有 Microsoft 計畫的整備素材。 我們將在 Microsoft 合作夥伴網站 ([https://partner.microsoft.com](https://partner.microsoft.com)) 上推出一個作業整備資源位置，以合併來自[雲端解決方案提供者計畫更新資源庫](https://partner.microsoft.com/resources#/)及 [Microsoft 整備網站](https://www.microsoftreadiness.com/)的資產。

在將作業整備資源齊聚到單一位置後，合作夥伴就可以比以往更加輕鬆地尋找所有 Microsoft 管道和計畫的最新更新、重大公告與重要的作業整備資源。 透過單一登入體驗和增強的搜尋、篩選與排序功能，合作夥伴可以輕鬆地存取、取用和準備其業務，以適合的方式成功地與 Microsoft 交易和往來。

### <a name="expanding-on-the-success-of-the-csp-program-updates-resource-gallery"></a>詳述 CSP 計畫更新資源庫的成就：

partner.microsoft.com 上的作業整備資源位置在運作時所採用的方式，很像目前的 CSP 計畫更新資源庫，但會有一些顯著的增強和修改，包括：

- **計畫頁面：** 透過已預先篩選為只剩下該計畫相關內容的頁面，尋找特定計畫的最新文件更新會變得很容易。
- **功能強大的資源庫搜尋：** 內容篩選器可讓您更快速且更輕鬆地尋找所有計畫的內容，例如 CSP。
- **公告：** 透過已預先篩選為只剩下該計畫相關內容的頁面，尋找特定計畫的最新公告和最新文件更新會變得更加容易。 CSP 也會有一個計畫頁面，但合作夥伴中心仍會繼續張貼 CSP 公告。
- **新增位置：** 在 partner.microsoft.com 上用來存放作業資源的新位置上線後，您就無法再存取 Microsoft 整備網站和 CSP 計畫更新資源庫，且這兩個資源會將使用者重新導向至新的作業資源位置。 合作夥伴將能夠利用內容計畫篩選器來篩選出 CSP 或其他特定計畫的資源。
- **內容已擴大：** 來自 Microsoft 整備網站和 CSP 計畫更新資源庫的所有最近和相關內容，都會重新放到新的作業準備位置，這表示現在會有更多計畫的內容放在同一個位置。

## <a name="microsoft-365-business-voice-available-to-csp-partners-from-april-1-2020"></a><a id="2"/></a>自 2020 年 4 月 1 日起，CSP 合作夥伴已可開始使用 Microsoft 365 Business Voice

### <a name="impacted-audience"></a>影響對象

- 在這些[國家/地區](https://partner.microsoft.com/resources/collection/microsoft_365_business_voice_in_csp#/)營運的所有雲端解決方案提供者合作夥伴。

### <a name="details"></a>詳細資料

### <a name="customer-value-proposition"></a>客戶價值主張

Microsoft 365 商務語音是一種雲端式電話系統，適用於最多 300 位使用者的企業。 其可讓您使用 Microsoft Teams 中的任何電話號碼撥打電話並接收來自 Microsoft Teams 中任何電話號碼的通話。 搭配 Teams，即可提供結合通話、聊天和會議的多功能通訊解決方案。

Microsoft 365 商務語音將生產力和通訊簡化並整合成一個應用程式，不論是在電腦、智慧型手機還是電話機上，凡處理業務之處均可運作。 Microsoft 365 商務語音的優點包括：

- 多功能通訊。 整合通話、聊天、會議、行事曆和電子郵件，徹底改變完成工作的方式。
- 輕鬆且智慧化的通話。 以更有彈性、輕鬆且智慧化的方式通訊，讓您可以隨時保持連絡。
- 由 Microsoft 雲端提供。 透過可靠且安全的雲端式解決方案，安心地工作。

Microsoft 365 商務語音是以下列方案的附加元件形式提供：

- Office 365 商務基本版、商務進階版、F1、A1、E1、A3、E3
- Microsoft 365 商務版、F1、A3、E3

合作夥伴可以透過以下方式利用此商機創造營收：

- 啟用具有通話佇列/自動語音應答功能的電話系統、評估網路品質，或設定直接路由和音訊會議。
- 提供使用者採用/訓練服務，以確保組織的整備程度，並開始使用新式通訊工具 Microsoft Teams。
- 使用品質報告、裝置管理和使用者支援等受控服務，支援及監視客戶環境。

## <a name="act-now-partners-are-now-required-to-have-accepted-the-microsoft-partner-agreement"></a><a id="1"/></a>立即行動：合作夥伴現在必須接受 Microsoft 合作夥伴合約。

### <a name="categories"></a>類別

- 日期：2020-04-01
- 輸入：功能
- 狀態：現已推出
- [深入了解](https://partner.microsoft.com/resources/collection/new-microsoft-cloud-agreement-csp)

### <a name="impacted-audience"></a>影響對象
所有雲端解決方案提供者合作夥伴。

**2020 年 4 月 1 日更新** 截自 2020 年 1 月 31 日止，雲端解決方案提供者 (CSP) 計畫的所有合作夥伴均應已接受其所有交易雲端解決方案提供者租用戶的合約。 我們先前曾通知，該合約下一個實施階段時間將於 2020 年 4 月開始。  為了協助您避免業務中斷，我們已決定將下一個實施階段時間延長到 2020 年 4 月之後。 有些雲端解決方案提供者合作夥伴向我們反應，他們接受 MPA 時發生問題。 因為發生這些情況，所以我們正努力確保 Microsoft 資源能夠支援在接受合約時發生問題的合作夥伴。

### <a name="summary"></a>摘要
雲端解決方案提供者計畫中所有合作夥伴 (包括間接轉銷商) 都必須在 2020 年 1 月 31 日之前接受 Microsoft 合作夥伴合約，以確保其可繼續在雲端解決方案提供者計畫中建立新客戶或為現有客戶下新訂單。 雲端解決方案提供者計畫中所有合作夥伴現在都必須接受 Microsoft 合作夥伴合約。

自 2020 年 4 月起：
- 未接受 Microsoft 合作夥伴合約的直接計費合作夥伴和間接提供者，將無法下新訂單，只能管理現有的訂用帳戶。
- 未接受 Microsoft 合作夥伴合約的間接轉銷商，將不再有資格擔任新訂單的記錄合作夥伴。

**如果您尚未接受，請立即完成下列動作。**

### <a name="actions-for-direct-bill-partners"></a>直接計費合作夥伴需執行的動作：
1. 通知組織中的主要利害關係人檢閱 Microsoft 合作夥伴合約。
2. 使用全域系統管理員認證，在[合作夥伴中心的雲端解決方案提供者 [概觀] 頁面](https://partner.microsoft.com/pcv/dashboard/overview)上接受 Microsoft 合作夥伴合約。

如需詳細資訊和資源，請參閱[這裡](https://partner.microsoft.com/resources/collection/new-microsoft-cloud-agreement-csp#/)的 [CSP 資源庫] 頁面。

### <a name="actions-for-indirect-providers"></a>間接提供者需執行的動作
1. 通知組織中的主要利害關係人檢閱 Microsoft 合作夥伴合約。
2. 使用全域系統管理員認證，在[合作夥伴中心的雲端解決方案提供者 [概觀] 頁面](https://partner.microsoft.com/pcv/dashboard/overview)上接受 Microsoft 合作夥伴合約。
3. 使用 [API 來驗證間接轉銷商的簽署狀態](/partner-center/develop/verify-indirect-reseller-mpa-status)。
4. 協助間接轉銷商在下方完成其動作。

如需詳細資訊和資源，請參閱[這裡](https://partner.microsoft.com/resources/collection/new-microsoft-cloud-agreement-csp#/)的 [CSP 資源庫] 頁面。

我們知道間接提供者測試交易強制功能時，所遇到的技術問題。 這些問題將會在 2020 年 4 月解決。

### <a name="actions-for-indirect-resellers"></a>間接轉銷商需執行的動作
1. 以間接轉銷商身分[上線到合作夥伴中心](https://partner.microsoft.com/resources/collection/indirect-reseller-onboarding-for-mpa-in-csp#/)，並在上線期間接受 Microsoft 合作夥伴合約。
2. 若您先前已使用間接轉銷商身分上線到合作夥伴中心，請使用全域系統管理員認證在[合作夥伴中心儀表板頁面](https://partner.microsoft.com/pcv/dashboard/overview)接受 Microsoft 合作夥伴合約。
如需間接轉銷商專屬資訊和資源，請參閱[這裡](https://partner.microsoft.com/resources/collection/indirect-reseller-onboarding-for-mpa-in-csp#/)的 [轉銷商 CSP 資源庫] 頁面。
