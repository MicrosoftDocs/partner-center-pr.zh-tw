## <a name="changes-to-enterprise-voice-and-audio-conferencing-extended-dial-out-launches"></a><a name="10"></a>對企業語音和音訊會議擴充撥出啟動的變更

- 日期：2020-07-31
- 供應項目/市場

### <a name="impacted-audience"></a>影響對象

直接提供者、間接提供者和間接轉銷商。

## <a name="changes-coming-to-partner-center-apis"></a><a name="9"></a>合作夥伴中心 API 的變更

### <a name="categories"></a>Categories

- 日期：2020-07-28
- 功能

### <a name="impacted-audience"></a>影響對象

使用合作夥伴中心 API 且參與 CSP 計畫的 Direct Bill 合作夥伴和間接提供者。 

### <a name="summary"></a>摘要

合作夥伴中心小組評估了公開的合作夥伴中心 API。 我們看到其中一些 API 已過時，並且不再受到支援。 這些 API 將於 2020 年 8 月 31 日淘汰。

### <a name="details"></a>詳細資料

CSP 合作夥伴中心小組已找出計畫在 2020 年 8 月 31 日結束時淘汰的幾個 API。 這些 API 是多年前設計的介面，其中有許多介面已不再運作。 小組將在下一個 SDK 版本 (將於 2020 的最後一個行事曆季度發行) 中，移除這些端點的參考。 以下是要淘汰的 API。 合作夥伴應確保他們之後不會參考這些 URI。

**合作夥伴服務要求** - 這些端點可讓合作夥伴取得支援主題字串，然後以程式設計方式提交合作夥伴服務要求。 隨著時間過去，此 API 已變得不穩定且不受支援。 系統會將合作夥伴導向合作夥伴中心的使用者介面，以建立合作夥伴支援票證。 使用者體驗會在建立支援案例時提供其他資訊，例如建議的步驟，以及問題所在領域的相關文件。 合作夥伴中心小組也自訂了服務要求表單來擷取問題領域特有的資訊，來增強合作夥伴票證的建立體驗。 線上表單所收集的資訊可協助支援工程師更快速且準確地解決您的問題。 您不應該參考或依賴這些 URI，因為這些 URI 將在 2020 年 8 月底時淘汰。 請注意，這些變更只會影響取得支援主題和建立**合作夥伴**服務要求的方法，所有其他與支援相關的 API 應會繼續正常運作。

- {base-URL}/v1/servicerequests/supporttopics
- {base-URL}/v1/servicerequests/{agent-locale}

**Office 服務事件** - 這些端點旨在讓合作夥伴能夠在其所有合作夥伴的服務上，取得服務事件/中斷的彙總觀點。 不過，這些端點傳回的資料已過時且不實用。 我們將在 7 月底淘汰這些端點。 接下來，合作夥伴中心小組將重新評估「所有客戶」的服務事件和健康情況案例，並尋找在此領域中增加價值的機會。 這些 URI 的參考都應移除。

- {base-URL}/v1/serviceincidents/office/count
- {base-URL}/v1/serviceincidents/office

合作夥伴可以使用 [Office 365 服務通訊 API](/office/office-365-management-api/office-365-service-communications-api-reference) 來取得客戶服務的最新狀態。

**TopProduct** - 此端點已建立為不適合正式發行的概念證明。 其設計目的是讓合作夥伴管理在合作夥伴中心目錄中呈現的熱門供應項目。 不過，此 API 未曾完整地開發和完成，並且將在 2020 年 7 月 31 日前停用。 合作夥伴應移除此 API 的任何參考。

- {base-URL}/v1/preferences/catalog/products

### <a name="next-steps"></a>後續步驟

- 合作夥伴應確保其解決方案不再參考上述 API。
- 如有需要，「雲端解決方案提供者」合作夥伴可以造訪[合作夥伴中心 SDK 和 API Yammer 社群](https://www.yammer.com/cloudpartnercommunity/#/Threads/show?threadId=786951767490560)來進一步釐清問題。

## <a name="sdk-release-on-net-standard-v1161-preview"></a><a name="8"></a>.NET Standard 上的 SDK 版本 (v1.16.1-preview)

### <a name="categories"></a>類別

- 日期：2020-07-21
- 功能

### <a name="impacted-audience"></a>影響對象

使用合作夥伴中心 .NET SDK 且參與 CSP 計畫的 Direct Bill 合作夥伴和 Indirect Provider。 

### <a name="summary"></a>摘要

自 2020 年 7 月 21 日起，合作夥伴可以開始下載 **.NET Standard SDK Microsoft 合作夥伴中心**的預覽版本 (.NET SDK v1.16.1-preview)，以及更新的公用**合作夥伴中心 SDK GitHub 範例**。

### <a name="details"></a>詳細資料

參與 CSP 計畫的 Direct Bill 合作夥伴和 Indirect Provider 都有資格下載最新的 .NET Standard SDK 預覽版本。 我們會將 Microsoft 合作夥伴中心 SDK 從 .NET Framework 遷移至 .NET Standard 2.0 平台。 這些變更會使用 .NET Framework 4.6.1 和更新版本，讓 SDK 與現有的應用程式相容。 .NET Standard 將支援 .NET Core 2.0 和更新版本。

已有大量合作夥伴要求使用 .NET Standard 的 SDK。 目前的 SDK 版本會強制合作夥伴使用以 ASP.NET Framework 撰寫的應用程式，但此方式比 .NET Core 應用程式緩慢且成本高昂。 因此，遷移至 .NET Standard 有助於解決 .NET Core 的效能或相容性問題。
 
**重點**
- Microsoft 不會再針對 .NET Framework 4.8 提供其他版本，因為這是其產品線中的最後一版。
- 未來的開發會受到限制，因為大部分的程式庫會針對 .NET Core 或 .NET Standard 來發行，而這可能不支援目前的架構版本。
- 由於 .NET Core 的輕量本質，其很適合與容器搭配使用。
- 適用於 Windows Server 和 Linux 的高效能伺服器執行階段會使 .NET Core 成為最佳的 Web 架構。

### <a name="next-steps"></a>後續步驟

- 下載最新版本的 [.NET SDK v1.16.1-preview](https://www.nuget.org/packages/Microsoft.Store.PartnerCenter/1.16.1-preview)
- 版本：.NET SDK v1.16.1-preview 
- 範例程式碼：[範例程式碼](https://github.com/microsoft/Partner-Center-DotNet-Samples)
- 預覽版發行時間：7/21/2020 
- GA：預覽版發行之後三週 (為提供足夠的時間讓合作夥伴進行移轉) 
- 目前版本中沒有新的 API 變更

## <a name="cloud-solution-provider-csp-incentive-for-eligible-microsoft-commercial-marketplace-offers-enhanced-for-fy21"></a><a name="7"></a>適用於合格 Microsoft 商業市集供應項目的雲端解決方案提供者 (CSP) 獎勵在 FY21 獲得提升

### <a name="categories"></a>類別

- 日期：2020-07-22
- 功能

### <a name="impacted-audience"></a>影響對象

參與 CSP 計畫的 Direct Bill 合作夥伴和 Indirect Provider

### <a name="summary"></a>摘要

自 2020 年 7 月 1 日起，CSP Direct Bill 合作夥伴和 Indirect Provider 可以在符合資格的基礎結構即服務 (IaaS) 供應項目的交易上獲得獎勵。

### <a name="details"></a>詳細資料

參與 CSP 計畫的 Direct Bill 合作夥伴和 Indirect Provider 皆有資格在 [Microsoft 商業市集](https://azure.microsoft.com/overview/commercial-marketplace)中，為其客戶交易精選的第三方供應項目。 作為提高商業市集收益的獎勵，合格的 CSP Direct Bill 合作夥伴和 Indirect Provider 可在交易合格供應項目時獲得獎勵。

自 2020 年 7 月 1 日起，您可以在合格 IaaS 供應項目的交易上獲得獎勵。 為合格軟體即服務 (SaaS) 供應項目提供的獎勵會繼續在 FY21 中進行。

您可以在 [Microsoft 合作夥伴網站](https://aka.ms/partnerincentives)找到符合獎勵資格的最新 SaaS 和 IaaS 供應項目清單。 符合資格的供應項目清單會每月更新，並張貼至 CSP Direct Bill Partner 獎勵計畫和 CSP Indirect Provider 獎勵計畫的資產集合。 CSP 合作夥伴可以透過合作夥伴中心或 Azure 入口網站來起始這些交易。

若要獲得獎勵，您必須符合相關獎勵計畫指南中所述的所有資格和註冊需求。 [合作夥伴網站](https://aka.ms/partnerincentives)會提供計畫指南和其他實用的獎勵資源。

### <a name="next-steps"></a>接下來的步驟

- 請參閱完整公告，並深入了解作業整備資源庫集合中的商業市集：[合格 Microsoft 商業市集供應項目的 CSP 獎勵](https://partner.microsoft.com/resources/collection/csp-incentive-for-microsoft-commercial-marketplace-offers#/)。
 
- 請在每個月初，檢查合格的 SaaS 和 IaaS 供應項目清單。
 
- 與客戶合作，以在 CSP 計費關係中提供這些服務。

## <a name="new-capabilities-added-to-the-insights-reports-in-partner-center"></a><a name="6"></a>在合作夥伴中心的深入解析報告中新增功能 


### <a name="categories"></a>Categories

- 日期：2020-07-16
- 功能
 
 
### <a name="impacted-audience"></a>影響對象

Microsoft 合作夥伴網路中的所有合作夥伴組織 

### <a name="summary"></a>摘要

已將新功能新增至合作夥伴中心的深入解析報告，讓合作夥伴能夠根據其商務需求執行臨機操作資料分析，並查看員工在組織中接受培訓的整體觀點。  這些新功能會增強[現有深入解析報告](../partner-center-insights.md)的穩固性。

新功能概觀：

- **原始資料匯出**啟用了客戶詳細資料、訂用帳戶詳細資料、Microsoft Azure 使用方式、Office 365 使用方式、Dynamics 365 使用方式、企業行動力與安全性 (EMS) 使用方式、Power BI 使用方式、訓練和認證以及 Microsoft Learn。 如需詳細資訊，請參閱[這篇文章](../pci-download-reports.md)。

- **新的訓練儀表板**可提供員工在組織中接受培訓的整體觀點。 如需詳細資訊，請參閱[這篇文章](../pci-training-dashboard.md)。

### <a name="what-is-insights"></a>什麼是深入解析？

合作夥伴中心的深入解析儀表板可以橫跨雲端產品 (例如 Office 365、Azure、Dynamics 365) 以及授權模型 (例如雲端解決方案提供者 (CSP) 和 Enterprise 合約 (EA))，為您提供合作夥伴組織的關鍵效能指標 (KPI) 統一而全面地觀點。 Intelligent Insights 會提供給您，以讓您清楚了解您與 Microsoft 的業務往來，並放心根據資料制定決策。   

若要查看此儀表板，請從合作夥伴中心的左側導覽功能表中，選取 [深入解析]。 此儀表板使用與目前 MyInsights 報告工具相同的基礎資料，但提供更豐富的視覺效果，以及更深入的報告功能和 KPI，範圍涵蓋客戶、訂用帳戶、授權使用方式、專長認證和權益。 請經常回來查看，因為我們會持續為深入解析儀表板帶來新的報告和功能。    

### <a name="next-steps"></a>接下來的步驟

我們建議您使用 [深入解析] 索引標籤中所提供的內嵌意見反應工具，探索這些新功能並分享您的意見。

### <a name="impacted-audience"></a>影響對象

CSP 間接轉銷商和合格的 CSP 間接提供者 

### <a name="details"></a>詳細資料

我們致力於在 Microsoft 透過其數位轉型旅程來支援每個組織。 我們的作法不僅是投資技術，也藉由檢查從獲取我們的技術到其支援體驗的端對端客戶和合作夥伴旅程。 透過這項工作，我們已藉由投資我們的商務平台，實現自我轉型的機會。  我們也在發展全新商務體驗的同時簡化授權計畫，讓我們持續改善為合作夥伴提供更多商機的方式。 

隨著我們改善了商業功能以為客戶和合作夥伴提供更好的購買體驗，我們擴充了供應項目組合，以在雲端解決方案提供者方案 (CSP) 內包含永久軟體授權的可用性。 在 CSP 內新增軟體授權，讓參與的合作夥伴可以專注於為客戶提供各種解決方案和受控服務。 其也使客戶能夠轉換至雲端，並為合作夥伴提供混合式雲端環境所需的彈性。   

自 7 月 1 日起，我們將透過一組合格的間接提供者及其間接轉銷商來加速這項功能，其能夠與 CSP 計畫中的客戶交易內部部署軟體的授權。 合格的合作夥伴是由 Microsoft 所管理，而且必須同時是作用中的 CSP 間接提供者和 Microsoft 授權的經銷商 (適用於目前支援 CSP 內軟體銷售地區中作業的 Open License 方案)。 自 2021 年 1 月開始，Microsoft 將進一步調整這項功能，並讓所有 CSP 直接帳單合作夥伴、間接提供者和間接轉銷商在 CSP 計畫內銷售軟體授權。
