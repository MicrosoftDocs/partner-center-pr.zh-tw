# <a name="how-indirect-providers-can-help-their-indirect-resellers-accept-the-microsoft-partner-agreement-mpa"></a>間接提供者可以如何協助間接轉銷商接受 Microsoft 合作夥伴合約 (MPA) 

**適當的角色**

- 全域系統管理員
- 系統管理代理人
- 銷售代理人

下列問題和解答解決了可能的問題和解決方法，協助間接轉銷商瞭解如何及為何接受 MPA。 

雲端解決方案提供者計畫中的間接提供者、直接帳單夥伴和間接轉銷商將會發現這項資訊很有用。

## <a name="questions-on-mpa-program-and-policy"></a>MPA 程式和原則的相關問題

#### <a name="what-are-the-legal-implications-if-the-indirect-reseller-does-not-accept-the-mpa"></a>如果間接轉銷商不接受 MPA，會有哪些法律含意？
 
**解答** 除非您 (間接轉銷商) 接受 MPA，否則您將無法以 CSP 方案中的間接提供者 (散發者) 進行交易。 您必須針對您想要在其中進行業務的每個 [CSP 區域](regional-authorization-overview.md) 接受 MPA。 針對間接轉銷商，我們需要至少一個每個 CSP 區域的 MPA 簽章，才能符合使用該 CSP 區域中任何間接提供者的 CSP 交易。 

#### <a name="is-mpa-available-in-local-languages"></a>MPA 是否以當地語言提供？
 
**解答** 不。 它只提供英文版。 

#### <a name="my-reseller-believes-they-are-already-enrolled-as-an-indirect-reseller-because-they-have-been-able-to-resell-cloud-subscriptions-through-us-however-they-have-no-option-to-accept-the-mpa-why-is-that"></a>我的轉銷商認為他們已經註冊為間接轉銷商，因為他們已可透過美國轉售雲端訂用帳戶，但沒有接受 MPA 的選項。 這是為什麼？

**解答** 如果轉銷商可以登入合作夥伴中心但未看到接受 MPA 的選項，或確認它們已在 CSP 間接轉銷商計畫中註冊，則很可能表示他們只在 MPN 方案中註冊，而不是在 CSP 中註冊為間接轉銷商。 在過去，我們已允許轉銷商透過其間接提供者進行交易，即使他們未正式註冊 CSP 方案也一樣。

現在是時候成為官方的 CSP 間接轉銷商。
這也適用于已在 CSP 中交易的「Advisor」合作夥伴。 Microsoft 在企業的每個 CSP 區域都必須有作用中的 CSP 間接轉銷商帳戶，才能讓間接轉銷商與間接提供者交易。

#### <a name="i-work-with-multiple-indirect-providers-distributors-in-various-countries-do-i-need-to-accept-the-mpa-in-each-csp-country"></a>我使用多個間接提供者 (散發者) 在不同的國家/地區。 我是否需要在每個 CSP 國家/地區接受 MPA？


**解答** 針對 MPN 識別碼，每個 CSP 區域至少需要一個 MPA 簽章。 例如，如果您在法國有 MPN 識別碼，並已在法國和已接受的 MPA 註冊至 CSP 間接轉銷商方案，則您在整個 [CSP 歐洲區域](regional-authorization-overview.md#europe-region-and-market) 都符合規範，而且不需要接受其他歐洲國家/地區的 MPA。 您可以使用整個 CSP 歐洲區域中的任何間接提供者來交易。 

#### <a name="i-am-a-direct-bill-partner-in-france-and-also-transact-with-an-indirect-provider-in-italy-do-i-need-to-accept-two-mpas--one-as-direct-bill-and-one-as-indirect-reseller-"></a>我是法國的直接帳單合作夥伴，而且也與義大利的間接提供者合作，我是否需要接受兩個 MPAs，一個是直接帳單，另一個則是間接轉銷商？

**解答**  如果您已在 CSP 區域中接受直接帳單 MPA，則不需要接受 CSP 間接轉銷商 MPA。 不過，您必須確定您的 MPN 帳戶位於合作夥伴中心，且與間接提供者共用的 MPN 識別碼合作夥伴中心。 

#### <a name="do-we-have-to-keep-accepting-the-indirect-reseller-mpa-agreement-every-year"></a>我們每年都必須持續接受間接轉銷商的 MPA 合約嗎？


**解答** 您不需要每年接受。 若要與間接提供者進行交易，必須接受至少一個每個 CSP 區域的 MPA，才能維持合規性。


#### <a name="if-the-subscriptions-are-not-changing-does-it-matter-if-resellers-accept-the-agreement"></a>如果訂用帳戶未變更，轉售商是否接受合約會有什麼關係？ 

**解答** 如果間接轉銷商不接受 MPA，系統將會使用間接提供者來封鎖它們交易。 他們的 Microsoft 獎勵將會受到影響。 

#### <a name="will-i-be-off-boarded-from-partner-center-if-i-dont-accept-the-mpa-what-are-the-exact-risks-if-resellers-dont-accept-the-mpa"></a>如果我不接受 MPA，我會從合作夥伴中心關閉上線嗎？ 如果轉售商不接受 MPA，會有哪些確切的風險？

**解答** 間接轉銷商將使用間接提供者 (散發者) 來封鎖交易。 在不久的將來，將不會有間接轉銷商的計畫。 

#### <a name="my-reseller-says-that-the-mpa-was-already-accepted-by-an-employee-who-left-the-company-do-they-need-to-accept-again"></a>「我的轉銷商」指出 MPA 已被離職的員工接受。 是否需要再次接受？

**解答** 不

#### <a name="my-reseller-recently-changed-company-name-but-accepted-the-mpa-already-do-they-need-to-accept-again"></a>我的轉銷商最近變更了公司名稱，但已接受 MPA。 是否需要再次接受？

**解答** 不

#### <a name="do-we-have-to-communicate-with-our-resellers-about-accepting-the-mpa-or-microsoft-will-work-with-resellers-directly-to-accept--mpa"></a>我們必須與轉銷商討論是否接受 MPA，否則 Microsoft 將直接與轉售商合作接受 MPA？
 
**解答** 身為間接提供者，您必須負責清楚地與所有間接轉銷商溝通，並讓他們知道不接受 MPA 的後果。 您也必須提供接受 MPA 的適當支援和指引。 針對不是由您解決的重大問題，您應該使用此合作夥伴中心支援主題將它們重新導向至 Microsoft。

#### <a name="are-there-any-documents-that-describe-how-resellers-can-accept-on-the-mpa-and-confirm-if-the-latest-mpa-was-accepted"></a>是否有任何說明轉銷商如何接受 MPA 的檔，並確認是否接受最新的 MPA？ 

**解答** 以下是一些實用的公用連結，可協助間接轉銷商接受 MPA 並符合規範。 

1.  [適用于 CSP 的 Microsoft 合作夥伴合約 (MPA) ](microsoft-partner-agreement.md) (包含遷移 MPN 帳戶和接受合作夥伴中心中的 MPA 的逐步指引) 
2.  [與 CSP 計畫中的間接提供者合作](indirect-reseller-tasks-in-partner-center.md)
3.  [管理合作夥伴帳戶中的位置](manage-locations.md)

## <a name="questions-about-your-mpn-and-csp-account-structure"></a>MPN 和 CSP 帳戶結構的相關問題

#### <a name="what-is-the-difference-between-my-mpn-and-csp-accounts-and-how-are-they-related"></a>我的 MPN 和 CSP 帳戶之間有何差異？這些帳戶之間有何關聯性？

**解答** 您的 MPN 帳戶應該用來管理 Microsoft 的成員資格。 它可為您提供一個全域 MPN 識別碼 (PGA) ，以及一或多個位置 MPN 識別碼 (PLA) 取決於您所建立的子公司數目。 您的 CSP 帳戶會與您的位置 MPN 識別碼相關聯，並可用來與間接提供者進行交易或管理獎勵。

您與位置 MPN 識別碼相關聯的 CSP 帳戶，可讓您與間接提供者合作、管理客戶並獲得獎勵。
    
- [雲端解決方案提供者方案概觀](csp-overview.md)

- [雲端解決方案提供者 (CSP) 的 Microsoft 合作夥伴合約 (MPA)](microsoft-partner-agreement.md)

- [與 CSP 計畫中的間接提供者合作](indirect-reseller-tasks-in-partner-center.md)


#### <a name="what-is-the-difference-between-global-mpn-id-and-location-mpn-id"></a>全域 MPN 識別碼和位置 MPN 識別碼有何差異？

**解答** MPN Global 帳戶相當於您的總部，而 MPN 地點則是您的子公司。  有一個唯一的全域 MPN 識別碼 (PGA MPN 識別碼) ，以及一或多個唯一位置 MPN 識別碼 (PLA MPN 識別碼) 。 您可以使用 MPN 帳戶登入合作夥伴中心、選取 [ **設定**]、[ **帳戶設定**]，然後流覽至 [ **識別碼** ] 頁面，以檢查 MPN 識別碼。 

如果您的分公司位於不同的國家/地區，您可以在這些國家/地區建立 MPN 位置，並取得位置 MPN 識別碼。 然後，您可以使用這些位置 MPN 識別碼來註冊 CSP 方案、與間接提供者共用資訊，以及獲得獎勵。

#### <a name="i-dont-know-which-mpn-id-to-use-while-enrolling-into-csp-indirect-reseller-program"></a>我不知道在註冊 CSP 間接轉銷商計畫時要使用哪一個 MPN 識別碼？

**解答** 在 [註冊 CSP 間接轉銷商計畫](indirect-reseller-tasks-in-partner-center.md)之前，您的合作夥伴中心中應該有使用中的 MPN 帳戶。 MPN 識別碼 country 和您的 CSP 帳戶國家/地區應符合成功關聯，而 MPN 識別碼應為合作夥伴中心。 登入您的 MPN 帳戶 (MPN 和 CSP 登入認證可能不同) 。 移至 [**設定**  ->  **帳戶設定**  ->  **組織設定檔**  ->  **識別碼**]，並尋找符合您 CSP 帳戶國家/地區的 MPN 識別碼 **類型**"Location"。 在 CSP 註冊程式期間將 MPN 識別碼關聯至您的 CSP 帳戶之前，請先深入瞭解 [MPN 的位置](manage-locations.md) 。

#### <a name="how-do-i-check-which-mpn-id-is-associated-with-my-csp-account"></a>如何? 檢查哪些 MPN 識別碼與我的 CSP 帳戶相關聯？

**解答** 登入您的 CSP 間接轉銷商帳戶。 移至 [**設定**  ->  **帳戶設定**  ->  **組織設定檔**  ->  **識別碼**]。 在頁面的 CSP 區域下， **相關聯的 MPN id** 是與您的 csp 帳戶相關聯的 MPN 識別碼，並應提供給您的間接提供者交易。 

#### <a name="how-can-i-view-all-my-mpn-ids"></a>如何查看我的所有 MPN 識別碼？

**解答** 使用 MPN 帳號憑證登入合作夥伴中心 (MPN 和 CSP 登入認證可能) 不同。 從 [**設定**] 選取 **帳戶設定**  ->  **組織設定檔**  ->  **識別碼** 以查看您的 MPN 識別碼。 若要加入新的 MPN 位置/子公司，請移至 [ **法律聲明設定檔** ]，然後按一下 [ **新增位置**]。

## <a name="question-about-working-in-partner-center"></a>合作夥伴中心中工作的相關問題

#### <a name="i-dont-know-who-the-partner-center-admin-is-who-can-accept-mpa-"></a>我不知道誰是誰可以接受 MPA 合作夥伴中心系統管理員？

**解答** 您公司中的全域管理員必須接受 MPA。 若要找出貴公司中被指派全域管理員角色的人員，請移至 [ **使用者管理]。** 在 [搜尋] 方塊中，輸入 global admin。將會提取指派全域管理員角色的人員名稱。 任何人都可以接受 MPA。 如果您的公司沒有全域管理員，請閱讀 [貴公司需要全域管理員時該怎麼辦](no-mpn-admin.md)。

:::image type="content" source="images/user-management-clean.png" alt-text="如何尋找全域管理員":::


#### <a name="my-reseller-says-that-in-partner-center-they-dont-see-an-option-to-accept-the-mpa-how-can-i-best-support-them"></a>我的轉銷商說，合作夥伴中心沒有看到接受 MPA 的選項。 我要如何才能獲得最佳支援？

**解答**  如果轉銷商可以登入合作夥伴中心但看不到接受 MPA 的選項：

1. 他們會在左邊的 CSP 索引標籤下看到 [總覽] 頁面嗎？

如果他們在 [合作夥伴中心] 功能表的 [CSP] 下看不到 [總覽]，表示尚未在 CSP 方案中註冊。 轉銷商將需要開始以 CSP 間接轉銷商的形式註冊，如同 [在 csp 方案中與間接提供者合作](indirect-reseller-tasks-in-partner-center.md)所述。 完成註冊之後，他們就能夠接受 MPA。

2. 如果他們看不到總覽，請要求他們移至 [**設定**  ->  **帳戶設定**  ->  **法律**  ->  **夥伴** 或 **轉銷商**] 索引標籤，並向下移至 [程式資訊]。 他們會在間接轉銷商狀態旁邊看到什麼狀態？


如果狀態是其他 **任何狀態，但作用** 中，則標記法律聲明商務設定檔驗證仍在進行中或已遭到拒絕。 在相同的頁面上，轉銷商必須檢查驗證狀態，然後按一下 [ **深入瞭解] 以修正此問題** ，轉銷商提供必要的檔以繼續驗證。

:::image type="content" source="images/active-status.png" alt-text="狀態為作用中":::

如果您的轉銷商需要通過法律聲明商務設定檔檢查的支援，請要求他們開啟票證，選擇下列選項：

- 選取 [合作夥伴中心] 功能表上的 [ **支援** ]，然後選取 **[一般** > 帳戶驗證] > 然後選擇您需要協助的步驟。


#### <a name="how-do-i-know-if-i-have-right-role-in-partner-center-to-accept-mpa"></a>如何? 知道我是否有合作夥伴中心接受 MPA 的正確角色？

**解答** 只有全域管理員可以接受 MPA。 若要瞭解您在合作夥伴中心中的角色：

1. 登入合作夥伴中心中的 CSP 間接轉銷商帳戶，並先確認您已註冊 CSP 間接轉銷商帳戶。 

2. 移至 [ **使用者管理** ]，並檢查已登入的帳戶是否已獲指派「全域管理員」角色？ 如果不是，則不能用來接受 MPA。 在 [使用者管理] 頁面的 [搜尋] 方塊中搜尋全域管理員，並與他們聯繫以接受 MPA。

:::image type="content" source="images/user-management-clean.png" alt-text="在合作夥伴中心中識別您的角色":::


#### <a name="how-do-i-check-if-my-company-has-accepted-the-mpa"></a>如何? 檢查我的公司是否已接受 MPA？

**解答** 使用 CSP 間接轉銷商帳戶登入合作夥伴中心。
如果您是 CSP 中的管理員代理程式或銷售代理程式，您可以移至 [合約] 頁面，確認 Microsoft 合作夥伴合約是可見的。 

:::image type="content" source="images/agreement-new.png" alt-text="在 [帳戶設定] 中尋找協定":::


如果您是全域管理員，請移至 CSP 總覽頁面。 如果您看到綠色橫幅，表示您已設定為使用該 CSP 區域中的間接提供者進行交易。 如果沒有，您會看到 [ **接受並繼續** ] 按鈕接受 MPA 並啟用您的 CSP 帳戶。

:::image type="content" source="images/banner-boxed.png" alt-text="綠色橫幅":::

#### <a name="i-want-to-accept-mpa-but-i-dont-see-any-place-button-to-accept-terms-to-do-so"></a>我想要接受 MPA，但是我沒看到任何地方 (按鈕接受詞彙) 這樣做呢？

**解答** 使用您的 CSP 間接轉銷商帳號憑證登入合作夥伴中心。 移至 CSP 總覽頁面。  如果您看不到 [ **接受] 和 [繼續** ] 按鈕，而且沒有綠色橫幅告訴您已接受 mpa，表示您沒有接受 mpa 的正確角色。 您必須洽詢您的全域管理員，才能接受 MPA。
