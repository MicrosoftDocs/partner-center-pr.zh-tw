# <a name="using-the-skills-report"></a>使用技能報表

**適當的角色**

- MPN 合作夥伴系統管理員
- 全域系統管理員

技能報表可讓 Microsoft 合作夥伴網路中的合作夥伴評估合作夥伴公司中的使用者所完成的技能。 報表中的技能可以包含合作夥伴技術使用者所採用的評量、認證或測驗。 

合作夥伴公司對這些類型技能的成就，是合作夥伴在特定 Microsoft 專長認證內取得銀級或金級狀態的其中一項需求。 合作夥伴利用專長來向目前和新客戶展示技術專長的領域。 它們也為達成這些夥伴的合作夥伴提供許多好處。 如需有關可用專長的詳細資訊及其優點，請 [參閱 Microsoft 合作夥伴網路專長](https://partner.microsoft.com/membership/competencies) 認證頁面。

貴公司的 MPN 合作夥伴系統管理員或全域管理員可能會想要執行與技能報表相關的下列工作：

- 定期檢查技能報表，以查看使用者技能的進度，以瞭解您公司對一或多個重要專長的達成進度。 當使用者離開您的公司時，您可能會失去相關的技能。 同樣地，當新的使用者加入時，您可能會得到一些技能。

- 規劃未來的技能訓練，以確保您的合作夥伴公司符合特定專長認證的技能需求。 這麼做可協助您的合作夥伴公司獲得新的銀級或金級專長認證，或維持現有的銀級或金級專長認證。 針對現有的銀級或金級專長認證，系統管理員應該在專長認證的週年日之前，檢查技能報告至少三個月，以找出專長認證是否達到/續約風險。 這個事先的警告可讓您的使用者取得新的技能。
- 確認您公司中的所有技術使用者都出現在技能報表中，且他們已使用他們的合作夥伴中心使用者帳戶，立即將其學習帳戶與其相關聯。 本文稍後的章節將說明如何進行。

- 與使用者合作，以確認其最新技能資訊在技能報表上的正確性。 如有需要，可協助使用者針對任何問題進行疑難排解。

## <a name="skill-report-prerequisites"></a>技能報告必要條件

您必須符合下列必要條件，才能查看技能報表，並開始套用公司技能以獲得專長認證：

- 請確定您已在 Microsoft 合作夥伴網路註冊。
 
   合作夥伴必須先在 Microsoft 合作夥伴網路註冊，才能取得專長認證或存取其使用者的技能報表。 如果您的公司還不是 Microsoft 合作夥伴網路的成員，請移至 [ [成員資格] 頁面](https://partner.microsoft.com/membership) ，然後選取 [ **立即加入**]。
- 請確定您在合作夥伴中心中具有適當的角色。
   只有被指派 MPN 合作夥伴系統管理員或全域管理員合作夥伴中心角色的夥伴公司中的使用者，才可以從合作夥伴中心 [儀表板](https://partner.microsoft.com/dashboard)查看技能報表。 深入瞭解合作夥伴中心中的可用 [使用者角色](permissions-overview or assign.md) 。 或者，請參閱如何 [建立或管理使用者帳戶](create-user-accounts-and-set-permissions.md) ，以將特定角色指派給使用者。
- 請確定貴公司的使用者在合作夥伴中心中也有自己的使用者帳戶。 
   使用者帳戶應該與您的夥伴公司相關聯。 瞭解如何 [建立或管理使用者帳戶](create-user-accounts-and-set-permissions.md)。
- 請確定貴公司的使用者將其技能或認證與其合作夥伴中心使用者帳戶相關聯。 
   使用者必須先確定他們的 [Microsoft Learning 帳戶 (MCP 識別碼) 與合作夥伴中心和合作夥伴大學相關聯](ms-learn-associate.md)，才能在合作夥伴中心技能報告中出現其技能或認證。 使用者遵循這些步驟之後，可能需要24到72小時的時間，技能才會出現在技能報表中。 最近加入您公司的任何新使用者也應完成這組步驟。

   > [!NOTE]
   > 在參加測驗或課程時，使用者應該使用他們在 MsCert、Microsoft Learn 或合作夥伴大學入口網站中使用的相同電子郵件地址。 如果使用者在其合作夥伴中心使用者帳戶中使用的電子郵件地址不同于與其相關聯的電子郵件地址，他們可能需要與他們的 MPN 帳戶管理員或全域管理員合作。

## <a name="skills-report-and-pmc-users"></a>技能報告和 PMC 使用者

如果您的合作夥伴公司已將其成員資格從 PMC 移至合作夥伴中心，您將會有六個月的 PMC 型技能會考慮到專長認證。 若要避免您的專長認證有風險，請下載技能報表，並務必識別並排列其資料指出的優先順序： 

- PMC 資料行中 **相關聯** 的 "Yes"

- [ **PC** ] 資料行中的 [否]
 
- 符合專長的 **相關 (選項) 或程式** 資料行中顯示的有效技能

如果您的使用者符合上述準則，請聯絡他們，要求他們儘快 [在合作夥伴中心中建立關聯](ms-learn-associate.md) 。

## <a name="troubleshoot-issues-with-the-skills-report"></a>針對技能報表的問題進行疑難排解

本節涵蓋您或您的使用者可能會遇到的技術報告問題，以及如何解決這些問題。

### <a name="why-cant-i-see-any-skills-for-a-specific-user"></a>為什麼我看不到特定使用者的任何技能？

如果技能報表資料對使用者不會顯示任何技能，使用者：

- 可能尚未取得相關技能

- 可能已將其技能合併到與合作夥伴中心使用者帳戶 **沒有** 關聯的另一個 MCID

如果使用者有重複的帳戶，則所需的技能可能會分割成兩個或更多帳戶。 例如，您可能會將 MSA 系結至 MCID (，如技能報表) 的 [MCID] 資料 **行** 所示，但文字記錄的相同 MSA 參考另一個 MCID。 在這種情況下，您必須將文字記錄中的 MCID 合併至已在技能報表中顯示的 MCID。 若要這樣做，請洽詢指定的 [Microsoft 認證訓練支援小組](https://trainingsupport.microsoft.com/mcp/forum)。 確認 MCP 合併之後，您就不需要重新建立 MSA 的關聯。 它會在合作夥伴中心中建立關聯。 此程式最多可能需要24小時的時間才會顯示在合作夥伴中心中。

### <a name="why-do-i-see-an-error-message-when-i-try-to-download-the-skills-report"></a>為什麼我在嘗試下載技能報表時看到錯誤訊息？

當您嘗試下載技能報表時，可能會看到下列訊息：「 *找不到具有技能資料的使用者。查看使用者是否已將他們的學習或夥伴大學帳戶與其設定檔相關聯。*

如果您的帳戶是在合作夥伴中心中直接建立，或您的公司的使用者在將其 Mcp 移至合作夥伴中心之前都沒有 PMC 相關聯的 PMC，就會出現此訊息。 
[查看與](ms-learn-associate.md) 您的使用者的學習或夥伴大學帳戶相關聯的 mcp 是否已經與合作夥伴中心中的使用者帳戶相關聯。

### <a name="a-user-has-associated-their-skills-in-partner-center-but-i-dont-see-it-in-the-skills-report"></a>使用者在合作夥伴中心中有相關聯的技能，但我在技能報告中看不到它

如果使用者已完成技能，但未出現在技能報表中，請確認：

- 使用者已在合作夥伴中心中建立相關技能的關聯

- 使用者的技能是特定專長認證需求的一部分 (如合作夥伴中心儀表板或 Microsoft 合作夥伴網路專長認證頁面) 所示。

- 已超過72小時，因為使用者在72小時之後完成其技能，如果技能資料仍未反映使用者技能的完成，MPN 帳戶管理員或 MPN 夥伴系統管理員應聯絡 [Microsoft 合作夥伴支援](https://partner.microsoft.com/support)。 在您聯絡 Microsoft 合作夥伴支援之前，請先準備好收集和傳送下列資訊：

- 已 **匯出的使用者報表的** 複本 (或報表) 的螢幕擷取畫面。 若要存取此報告，請登入合作夥伴中心，然後選取 [ **使用者管理**]，然後選取 [ **匯出使用者的報告**]。 報表會顯示誰可以存取 Partner 大學或誰有相關聯的 Microsoft Learning 帳戶」。 使用者也可以從合作夥伴中心儀表板右上角選取 [ **我的設定檔** ]，以提供這項資訊的螢幕擷取畫面。

- 如本文稍早所述，下載的最新技能報表版本。

- 證明使用者已完成相關的 Microsoft 測驗、認證或訓練和評量。 請參閱下列證明範例。

   > [!NOTE]
   > 目前，合作夥伴中心只會辨識來自兩個來源的憑證，Microsoft Learning 和合作夥伴大學。 從其他任何來源接收的任何其他憑證 (例如 OpenedX) ，將不予考慮。
