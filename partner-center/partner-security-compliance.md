### <a name="are-you-using-third-party-mfa-solution"></a><span data-ttu-id="f0ebb-198">您是否使用第三方 MFA 解決方案？</span><span class="sxs-lookup"><span data-stu-id="f0ebb-198">Are you using third-party MFA solution?</span></span>

<span data-ttu-id="f0ebb-199">如果您使用第三方 MFA 解決方案，請識別您要將其與 Azure AD 整合的方式。</span><span class="sxs-lookup"><span data-stu-id="f0ebb-199">If you are using third-party MFA solution, identify how you are integrating it with Azure AD.</span></span> <span data-ttu-id="f0ebb-200">一般來說有兩種方法，包括同盟和自訂控制項：</span><span class="sxs-lookup"><span data-stu-id="f0ebb-200">In general, there are two methods, including federation and custom controls:</span></span>

* <span data-ttu-id="f0ebb-201">**身分識別同盟**  - 當 Azure AD 收到驗證要求時，Azure AD 會將使用者重新導向至同盟識別提供者進行驗證。</span><span class="sxs-lookup"><span data-stu-id="f0ebb-201">**Identity federation** - When Azure AD receives an authentication request, Azure AD will redirect the user to the federated identity provider for authentication.</span></span> <span data-ttu-id="f0ebb-202">驗證成功之後，同盟識別提供者會連同 SAML 權杖，將使用者重新導向回 Azure AD。</span><span class="sxs-lookup"><span data-stu-id="f0ebb-202">Upon successful authentication, the federated identity provider will redirect the user back to Azure AD along with a SAML token.</span></span> <span data-ttu-id="f0ebb-203">為了讓 Azure AD 在向同盟識別提供者進行驗證時，辨識使用者已完成 MFA 驗證，SAML 權杖必須包含authenticationmethodsreferences  宣告 (具有 multipleauthn  值)。</span><span class="sxs-lookup"><span data-stu-id="f0ebb-203">In order for Azure AD to recognize that the user has completed MFA verification when authenticating to the federated identity provider, the SAML token must include the *authenticationmethodsreferences* claim (with value *multipleauthn*).</span></span> <span data-ttu-id="f0ebb-204">檢查同盟識別提供者是否支援發出這類宣告。</span><span class="sxs-lookup"><span data-stu-id="f0ebb-204">Check whether that the federated identity provider supports issuing such a claim.</span></span> <span data-ttu-id="f0ebb-205">若是如此，請檢查同盟識別提供者是否已設定為執行此動作。</span><span class="sxs-lookup"><span data-stu-id="f0ebb-205">If so, check whether the federated identity provider has been configured to do so.</span></span> <span data-ttu-id="f0ebb-206">如果宣告遺失，Azure AD (進而是合作夥伴中心) 將不會知道使用者已完成 MFA 驗證，而遺失宣告可能導致計量不是 100%。</span><span class="sxs-lookup"><span data-stu-id="f0ebb-206">If the claim is missing, Azure AD (and therefore Partner Center) will not know that the user has completed MFA verification and missing the claim can cause the metric to not be 100%.</span></span>
