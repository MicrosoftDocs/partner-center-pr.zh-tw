# <a name="view-manage-and-export-your-incentives-program-details"></a>查看、管理及匯出獎勵方案詳細資料

**適用於**

- 合作夥伴中心

**適當的角色**

- 獎勵系統管理員
- 獎勵使用者
- 獎勵唯讀檢視器
- 全域系統管理員
- MPN 合作夥伴系統管理員

使用這些頁面來查看您過去、暫止和即將推出的付款和收益的詳細資料、您在每個位置的獎勵計畫狀態，以及完整的交易歷程記錄。 您也可以下載或匯出任何交易的詳細資訊。 所有獎勵頁面都會顯示美元的金額。

>[!NOTE]
>您只會看到您所關聯之 MPN 識別碼和程式的資料。 如果您想要存取其他資料，請參閱您的獎勵系統管理員。 

## <a name="access-the-incentives-pages"></a>存取獎勵頁面

您可以從合作夥伴中心的 [儀表板](https://partner.microsoft.com/dashboard)中，透過幾種方式來存取獎勵資訊：

- 從左側功能表中選取 [ **獎勵** ]。 有幾個選項出現在 **獎勵**下，包括 **總覽**。 探索這些選項，以瞭解獎勵計畫、獎勵款項、收益和索賠的狀態。
- 選取右上角的 **Money** (支出) 圖示。 然後，您可以選取其他選項，例如 **[交易歷程記錄**]、[ **付款**]、[ **匯出資料**] 或 [ **辦公室銷售報告**]。

以下是此處所述的頁面：

- 我的獎勵總覽頁面
- [付款] 頁面
- [交易記錄] 頁面
- 匯出資料頁面和其他相關聯的匯出頁面
- 交易摘要報表

>[!NOTE]
>如需合作夥伴中心中獎勵和獎勵功能的詳細資訊，請參閱 [合作夥伴投資和獎勵](https://partner.microsoft.com/membership/partner-incentives) (登入所需的) 。

## <a name="my-incentives-overview-page"></a>我的獎勵總覽頁面

**我的獎勵概觀**會顯示獎勵計劃的整體狀態，以及每個程式在每個位置的狀態。 這項資訊包括您在獎勵計劃中註冊後的獎勵總收益和付款。

此頁面也包含收益和付款 (依位置或計劃)，以及建議您採取的所有進一步動作，以便在特定位置註冊計劃。

若要存取**我的獎勵概觀**：

1. 登入合作夥伴中心[儀表板](https://partner.microsoft.com/dashboard)。
2. 選取 [ **獎勵**]，然後從功能表中選取 **[總覽** ]。
3. 在頁面頂端查看收益和付款摘要，並在下表中取得進一步的詳細資料。 您也可以排序、分組和展開隨附的資料表：

   - 若要依資料行排序，請選取資料行名稱。
   - 若要依程式分組，請選取資料表上方的 [ **依程式** ] 索引標籤。
   - 若要依位置分組，請選取資料表上方的 [ **依位置** ] 索引標籤。
   - 若要查看特定群組內註冊的更多詳細資料，請選取指定資料列結尾的燕號符號。 這個箭號會展開您的觀點。

4. 如果需要進一步動作以註冊計劃，此資訊將會出現在**狀態**欄中。 在此情況下，請選取＞形箭號，以了解您需要採取的後續步驟。

## <a name="transaction-history-page"></a>[交易記錄] 頁面
此頁面會顯示所有的個人收益，包括日期、類型和每個項目的收益。 您可選取要檢視的時間間隔，也可透過註冊識別碼、計劃、付款識別碼、收益類型、控制桿和狀態等進行篩選。 資料適用于目前的會計年度 (7 月1日到6月30日) 和前兩個會計年度。

若要查看收益的詳細資料，請選取位於頁面右側的向下箭號。 這樣做會顯示 [杠杆]、[收入金額]、[產品] 和 [客戶]。 如果因為某些原因而無法使用這項資料，但您需要存取它，請聯絡支援人員。 如果收益是調整的結果，而不是交易，則不會顯示 [產品] 和 [客戶] 欄位。

若要匯出此頁面上的任何交易資料，請選取 [ **匯出** ]，然後依照 [匯出資料] 頁面上的指示進行。 從 [交易記錄] 頁面匯出的檔案會顯示交易貨幣中資料、兩種交易貨幣和美金收益，以及付款貨幣的付款值。

### <a name="estimated-payment-month"></a>預估的付款月份

[交易記錄] 頁面現在包含一個資料表，其中顯示您接下來幾個月的預估付款金額。 您也可以在交易歷程記錄和摘要報告匯出中，查看並下載此資訊。 這種資訊可讓您更輕鬆地進行對帳和付款預測。

預估的付款月份是根據程式設定規則和時程表來計算，並在下一個/即將進行的付款週期內進行處理。

預估的付款月份目前適用于所有的收入類型，但共同作業會顯示為 **不適用**。 若為2020年7月1日之前的收益，預估付款月份將會顯示為「 **無法使用**」。

#### <a name="estimated-payment-month-example"></a>預估的付款月份範例

| Month | 金額 |
| ------ | :-----------: |
|  Sep-2020 |  $7273.99   |
|  十月-2020 | $8692.30  |
|  11月-2020 | $107.89  |

預估數量可能會因各種原因而與實際數量不同：

- 贏得重述：如果已重新計算收益，則實際數量將會不同
- 調整：實際數量會根據發生或提交的調整而有所不同。
- 規則變更：規則中的變更可能會反映實際付款金額的重新計算
- 應付：如果發生付款失敗，則實際金額可能不同

請注意，如果符合您的程式的閾值和付款資格規則，則只會在投影月份釋出款項。 這些規則包括但不限於下列清單：

- 您的稅務設定檔必須是最新的
- 您的收益必須符合或超過您的節目表中定義的最小賺取閾值。
- 保留款項：如果您選取 [設定檔指派] 頁面上的 [保存我的付款] 選項。
- 無法使用付款檢測：付款或/和稅務設定檔未完成。

## <a name="export-data-page"></a>[匯出資料] 頁面
遵循此頁面上的指示，匯出您要的資料。

**注意：**

- 假設有大量的資料，所提交要求的處理 SLA 為24小時。
- 您只會看到您所關聯之 MPN 識別碼和程式的資料。 如果您想要存取其他資料，請參閱您的獎勵系統管理員。
- [匯出資料] 頁面不會自行重新整理。 您可能需要手動重新整理頁面，才能看到最新的資料。
- 篩選條件可能會導致**無可用資料**錯誤。 此錯誤可能表示您已將預設的時間週期保留在三個月，然後從該期間以外的收益中選取付款識別碼。 請展開時間間隔，然後再試一次。

## <a name="transaction-summary-report"></a>交易摘要報表

>[!NOTE]
>交易摘要報表將不會顯示您的所有程式或收益。 它是只著重于匯總拉杆的摘要。 

[交易摘要] 報表會針對匯總的收益杆提供摘要層級交易詳細資料，包括客戶名稱和訂用帳戶識別碼。 報表會牽涉到與多個交易相關聯的每個收益識別碼。 針對每個客戶/訂用帳戶，每個客戶/訂用帳戶會重複取得該收益識別碼的總收益金額和收益 這是因為匯總拉杆的獎勵是以統稱的，而不是每個訂用帳戶。


### <a name="download-the-transaction-summary-report"></a>下載交易摘要報表

1. 在 [ **獎勵總覽** ] 頁面上，選取右上角的 **Money** (支出) 圖示。
2. 選取 [匯出資料]。
3. 選取 [ **交易摘要** ] 索引標籤。
4. 選取適當的篩選器，以查看符合匯總收益的程式和拉杆。 如果未顯示任何資料，您的程式就不適合用于匯總的拉杆。
5. 選取 [**開始下載**]   以匯出報表。  

>[!NOTE]
>交易摘要報表將不會包含比60天新的訂用帳戶的客戶和訂用帳戶資訊，或者，如果耗用量收入小於1美分。
