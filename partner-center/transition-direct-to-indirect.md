# <a name="transition-from-cloud-solution-provider-csp-direct-bill-partner-to-csp-indirect-reseller"></a>從雲端解決方案提供者 (CSP) 直接帳單合作夥伴轉換到雲端解決方案提供者間接經銷商

**適用於：**

- 雲端解決方案提供者合作夥伴

>[!Note]
>本文適用于已決定轉型為間接轉銷商的直接帳單合作夥伴。 但是，即使您尚未做出明確的決策來註冊為間接轉銷商，但不符合 CSP 直接帳單合作夥伴方案新 [需求](direct-partner-new-requirements.md) 的直接帳單合作夥伴，將會在其 [直接帳單功能受到限制](restricted-direct-bill-capabilities.md)時收到通知。

您將能夠使用現有的直接帳單租使用者，註冊間接轉銷商計畫。

## <a name="get-started"></a>開始使用

1. 請確定合作夥伴中心和 MPN 識別碼中的夥伴設定檔都是最新的。

2. 以您要轉換為間接轉銷商的直接帳單租使用者的全域管理員身分登入合作夥伴中心。

   :::image type="content" source="images/direct/direct1.png" alt-text="概觀":::

3. 請參閱註冊表單上的合作夥伴詳細資料。

   :::image type="content" source="images/direct/direct2a.png" alt-text="立即註冊":::

4. 選取 [現在註冊]。 您的間接轉銷商企業將會使用您用於直接商務的相同 AAD 租使用者。

    > [!NOTE]
    > 一開始，這項新的轉換功能適用于9月到12月週年日期的合作夥伴。 如果您不是在9月到12月之間有週年日，此時將不會看到這項功能。 在2018年12月之後的交易夥伴，將會在啟用合作夥伴的功能之後通知稍後。

5. 核准您的註冊時，請再次登入合作夥伴中心。

    > [!NOTE]
    > 雖然核准通常是立即的，但最多可能需要五個工作天。 核准之後，您將會收到通知，告知您已在註冊表單中的 [主要連絡人] 下指定的電子郵件地址。 您也可以在 [**設定**  >  **夥伴設定**  >  **夥伴設定檔**] 之下，檢查您的註冊狀態 > 計畫資訊。

6. 在 **[總覽** ] 頁面上，您會看到間接轉銷商合約。 選取 **\[接受並繼續\]**。 此動作可啟用間接轉銷商功能。

當您已接受間接轉銷商合約時，請注意，您的夥伴設定檔會將您 **識別為直接** 帳單和間接轉銷商。

:::image type="content" source="images/direct/direct3.png" alt-text="間接轉銷商合約":::

> [!IMPORTANT]
> 當您使用新功能註冊為間接轉銷商後，就不會有任何選項可復原至直接帳單的租使用者。 在註冊為間接轉銷商之前，請務必完整評估您的商務需求。

## <a name="while-you-transition-from-direct-to-indirect-reseller"></a>當您從直接轉銷商轉換時

在這個階段中，您將會繼續管理直接客戶的訂用帳戶需求，包括計費流程。 您也可以開始接受來自間接提供者的客戶，並以間接轉銷商的形式運作。

:::image type="content" source="images/direct/direct4.png" alt-text="您是直接帳單和間接轉銷商":::

## <a name="find-an-indirect-provider"></a>尋找間接提供者

註冊之後，間接提供者的連結將會出現在左側導覽中。 身為間接轉銷商，您將建立與間接提供者的關聯性，該提供者可處理您的帳單、為客戶購買產品，以及支援基礎結構。

不同間接提供不同支援和服務，因此您應該評估當地提供者，以判斷何者最符合需求。 一般而言，大部分提供者會：

- 為您提技術訓練與協助
- 協助您行銷產品及服務
- 管理您的財務和點數條款

搜尋官方的 [Microsoft 間接提供者](https://partnercenter.microsoft.com/partner/find-a-provider)清單。

深入瞭解，  [與間接提供者的讀取合作夥伴](indirect-reseller-tasks-in-partner-center.md)

## <a name="accept-a-partnership-invitation-from-your-indirect-provider"></a>接受來自您間接提供者的合作邀請

當您找到與合作夥伴合作的間接提供者時，請在合作夥伴中心中建立與間接提供者的合作關係。

您選取的間接提供者會傳送電子郵件給您電子郵件，其中會帶您前往合作夥伴中心的邀請。 確定您的全域管理員登入合作夥伴中心，並遵循邀請連結。 當您接受邀請時，提供者的名稱將會出現在您的間接提供者清單中。

## <a name="acquire-new-customers-as-indirect-reseller"></a>以間接轉銷商的形式取得新客戶

您和間接提供者都必須與客戶建立轉銷商關聯性。 這些轉銷商關聯性可讓您代表客戶管理客戶的訂用帳戶和服務。 若要取得現有 Azure AD 租使用者的新客戶，您可以邀請客戶同時與您和您的提供者建立轉銷商關聯性。

若要建立間接轉銷商邀請：

1. 從您的合作夥伴中心左側導覽中選取 **間接提供者** 。

2. 選取 [邀請新客戶] 以邀請客戶同時與您及間接提供者建立經銷商關係。 提供者必須與您的客戶建立轉銷商的關聯性，讓他們可以在客戶想要購買新的訂用帳戶或將新的授權新增至現有的訂用帳戶時，代表客戶提交訂單。

3. 在下一頁，複查草稿電子郵件訊息。 您可以用電子郵件開啟草稿訊息，也可以將郵件複製到剪貼簿，然後貼到電子郵件中。

4. 編輯電子郵件中的文字以找出您需要的內容，但請務必包含該連結，因為它已個人化，可直接將客戶連接到您的帳戶和提供者的帳戶。 然後選取 [完成]****。

5. 客戶授權您和您的提供者成為其記錄可查經銷商之後，您就會有系統管理員權限，可代表他們管理其訂閱、授權和使用者，並且間接提供者可代表他們提交訂單。
6. 若要管理客戶的帳戶、服務、使用者及授權，請選取客戶名稱旁邊的向下箭頭以展開其記錄。

與直接帳單合作夥伴不同的是，間接轉銷商無法針對合作夥伴中心中的新客戶建立 Azure AD 的租使用者。 您的提供者會建立租使用者，並將您指定為此客戶的間接轉銷商。 這可確保客戶會出現在合作夥伴中心的客戶清單中。

>[!Note]
>您將無法使用直接帳單功能來為以間接轉銷商形式取得的客戶建立購買專案。

## <a name="managing-your-direct-bill-customers-and-your-indirect-reseller-customers"></a>管理您的直接帳單客戶和間接轉銷商客戶

您可以用不同的方式管理直接帳單客戶和間接轉銷商客戶。

### <a name="direct-bill-customers-things-you-wont-do-as-an-indirect-reseller"></a>直接帳單客戶 (不會做為間接轉銷商的專案) 

- 建立產品的訂單
- 管理 Azure 保留項目
- 管理其訂單歷程記錄
- 購買軟體
- 直接向客戶收費

### <a name="indirect-reseller-customers"></a>間接轉銷商客戶

- 您的間接提供者會為您的客戶訂購產品
- 管理客戶的授權和使用者
- 處理訂閱續約

### <a name="to-identify-customers-that-you-acquired-as-a-direct-bill-partner"></a>識別您取得為直接帳單合作夥伴的客戶

1. 選取 **客戶**

2. 選取要查看其詳細資料的客戶

3. 如果此客戶是您以直接帳單合作夥伴的形式取得，您將會看到 **新增** 或 **查看產品** 的選項，而且您會看到其訂用帳戶。

4. 如果客戶與您之間有間接轉銷商關係，將無法使用這些選項。

## <a name="move-your-direct-bill-customers-to-your-indirect-provider"></a>將您的直接帳單客戶移至您的間接提供者

您的間接提供者無法為現有的直接帳單客戶提交訂單或現有的訂用帳戶轉移，直到他們具有轉銷商關聯性為止。 若要在您的間接提供者與您現有的直接帳單客戶之間建立轉銷商關聯性，您可以使用下列其中一種方法：

- [轉銷商關聯性延伸模組](#reseller-relationship-extension)

- [將間接轉銷商邀請傳送給客戶](#send-an-indirect-reseller-invitation-to-the-customer)

您可以在「[直接到間接轉換」檔](https://partner.microsoft.com/resources/collection/Direct-Bill-transition-to-Indirect-reseller#/)中找到逐步程式的詳細總覽

### <a name="reseller-relationship-extension"></a>轉銷商關聯性延伸模組

您可以使用轉銷商關聯性延伸模組功能，利用合作夥伴中心儀表板，在您現有的直接帳單客戶與間接提供者之間建立轉銷商關聯性。 使用此功能之前，請注意下列事項：

- 這項功能只適用于正在轉換成為間接轉銷商的直接帳單合作夥伴完成 [間接轉銷商註冊](#get-started)。

- 您只能將這項功能套用至現有的直接帳單客戶。 不適用於 [間接轉銷商客戶](#acquire-new-customers-as-indirect-reseller)。

- 您只能選取已 [接受合作夥伴邀請](#accept-a-partnership-invitation-from-your-indirect-provider)的間接提供者。

- 您為此客戶提供的帳單寄送資訊複本將可供間接提供者使用。 您可以在合作夥伴中心儀表板中存取此客戶的帳戶頁面，以存取帳單資訊。

    > [!NOTE]
    > 使用轉銷商關聯性延伸功能，即表示您同意與間接提供者共用此客戶的帳單資訊。

- 您的間接提供者將不會提供給客戶租使用者的 [委派系統管理許可權](customers-revoke-admin-privileges.md) 。 如果您的間接提供者需要委派的系統管理許可權，您必須改為將間接轉銷商邀請傳送給客戶。

- 建立轉銷商關聯性之後，間接提供者將會在 [M365 系統管理中心](https://admin.microsoft.com/AdminPortal/Home#/partners) 和 [商務用 Microsoft Store](/microsoft-store/work-with-partner-microsoft-store-business)的 [合作夥伴關係] 頁面下，顯示為客戶的 CSP 合作夥伴。

    > [!IMPORTANT]
    > 為了避免混淆和誤解，您的合作夥伴合約會依履約不必，以在您使用關聯性延伸模組功能來建立現有直接帳單客戶與間接提供者之間的轉銷商關聯性之前，通知並取得直接帳單客戶的同意。

若要在現有的客戶租使用者上使用這項功能：

1. 以系統 **管理員代理程式**身分登入合作夥伴中心。

2. 在 [ **客戶] 頁面**中，選取現有的客戶，然後按一下其 [ **快速連結** ] 圖示，展開客戶的摘要視圖。

3. 在 [ **間接提供者 (s]) **底下，按一下 [ **將客戶轉移到間接提供者**]。

    :::image type="content" source="images/direct/direct5-1.png" alt-text="將客戶轉移至間接提供者":::

4. 在快顯對話方塊中，選取您想要與客戶的轉銷商關聯性的 **間接提供者** 。

5. 按一下 [儲存並繼續]。

6. 確認選取的間接提供者會顯示在 [ **間接提供者 (s]) **下。

    :::image type="content" source="images/direct/direct5-2.png" alt-text="列出的間接提供者":::

您的間接提供者不能送出現有直接帳單客戶的訂單，直到他們有轉銷商關聯性為止。 若要建立現有客戶與間接提供者之間的轉銷商關係，請使用間接轉銷商邀請邀請客戶。

1. 從您的合作夥伴中心左側導覽中選取 **間接提供者** 。

2. 選取 [邀請新客戶] 以邀請客戶同時與您及間接提供者建立經銷商關係。 提供者必須與您的客戶建立轉銷商的關聯性，讓他們可以在客戶想要購買新的訂用帳戶或將新的授權新增至現有的訂用帳戶時，代表客戶提交訂單。

    :::image type="content" source="images/direct/direct6.png" alt-text="邀請新客戶":::

3. 在下一頁，複查草稿電子郵件訊息。 您可以用電子郵件開啟草稿訊息，也可以將郵件複製到剪貼簿，然後貼到電子郵件中。

4. 編輯電子郵件中的文字以找出您需要的內容，但請務必包含該連結，因為它已個人化，可直接將客戶連接到您的帳戶和提供者的帳戶。 然後選取 [完成]****。

5. 客戶授權您和您的提供者成為其記錄可查經銷商之後，您就會有系統管理員權限，可代表他們管理其訂閱、授權和使用者，並且間接提供者可代表他們提交訂單。

6. 若要管理客戶的帳戶、服務、使用者及授權，請選取客戶名稱旁邊的向下箭頭以展開其記錄。

### <a name="microsoft-customer-agreement-acceptance"></a>接受 Microsoft 客戶合約

Microsoft Cloud 合約在2020年1月31日之前有效。 在該日期之後，所有客戶（現有和新的）都必須簽署新的 [Microsoft 客戶合約](confirm-customer-agreement.md)。 轉換客戶時，如果：

- **尚未接受客戶 Microsoft 客戶合約**

   請與間接提供者合作，讓客戶 [接受 Microsoft 客戶合約](confirm-customer-agreement.md)。

- **客戶已透過 Microsoft 365 系統管理中心接受 Microsoft 客戶合約**

   一旦與間接提供者建立轉銷商關聯性，就會保留接受。 您不需要執行任何動作。

- **客戶已透過合作夥伴證明接受 Microsoft 客戶合約**

   將不會保留接受。 請與間接提供者合作，以 [更新客戶在合作夥伴中心中的接受](confirm-customer-agreement.md#confirm-customer-acceptance-for-existing-customers)度。

## <a name="transfer-existing-direct-bill-subscriptions-to-indirect-provider"></a>將現有的直接帳單訂用帳戶傳送給間接提供者

在 CSP 間接模型下，間接轉銷商與 Microsoft 之間沒有帳單關聯性。 相反地，間接轉銷商會透過其間接提供者為其客戶取得訂用帳戶。 從直接帳單合作夥伴轉換為間接轉銷商時，您必須將您擁有的現有訂用帳戶轉移至間接提供者，作為直接帳單合作夥伴。 您可以使用合作夥伴中心儀表板中的自助式訂用帳戶轉移功能來執行這項操作。

### <a name="pre-requisites"></a>必要條件

- 這項功能僅適用于使用現有的直接帳單合作夥伴租使用者完成間接轉銷商註冊的轉換合作夥伴

- 在傳送與特定客戶相關聯的訂閱之前，轉換的夥伴必須將客戶移至間接提供者。

- 客戶必須 [透過間接提供者接受 Microsoft 客戶合約](#microsoft-customer-agreement-acceptance)。

### <a name="how-to-transition-to-indirect-reseller-status"></a>如何轉換為間接轉銷商狀態

這項功能是4個步驟的程式，其中：

- 轉換夥伴會建立訂用帳戶轉移要求。 要求包含一或多個與相同客戶相關聯的現有訂用帳戶，並且會被定址至間接提供者。

- 間接提供者會審核並接受 (或拒絕) 傳送要求。

- 間接提供者會確認傳送要求已完成。

- 轉換夥伴會確認傳送要求已完成。

> [!NOTE]

### <a name="transitioning-partner---create-transfer-request"></a>正在轉換夥伴-建立轉移要求

若要建立轉移要求作為轉換夥伴：

1. 以系統 **管理員代理程式**身分登入合作夥伴中心。

2. 在 [ **客戶** ] 頁面中，選取想要的客戶，然後按一下 [快速連結] 圖示，展開客戶的摘要視圖。

3. 在 [ **間接提供者 (s]) **下，確認已列出預期的間接提供者。

4. 按一下 [ **View 訂閱**]。

5. 在 [ **訂閱** ] 頁面中，尋找訂用帳戶 **轉移**。

6. 在 [ **訂閱轉移**] 下，按一下 [ **要求訂**用帳戶轉移]。

    :::image type="content" source="images/direct/direct8.png" alt-text="要求訂用帳戶轉移":::

7. 在 [轉移要求] 對話方塊中，選取要傳送的一個或多個訂閱。

    :::image type="content" source="images/direct/direct9.png" alt-text="建立轉移要求":::

8. 按一下 [建立]。

9. 作用中的訂用帳戶轉移要求會出現在訂用帳戶 **轉移**下。

    :::image type="content" source="images/direct/direct10.png" alt-text="傳送要求清單":::

10. 通知您的間接提供者您已建立訂用帳戶轉移要求。

### <a name="indirect-provider---accept-transfer-request"></a>間接提供者-接受轉移要求

若要以間接提供者的形式來檢查和接受轉移要求：

1. 以系統 **管理員** 代理程式或 **銷售代理程式**的身分登入合作夥伴中心。

2. 在 [ **客戶** ] 頁面中，選取想要的客戶，然後按一下其 [快速連結] 圖示，展開客戶的摘要視圖。

3. 在 [ **間接轉銷商 (s]) **下，確認已列出轉換的夥伴。

4. 按一下 [ **View 訂閱**]。

5. 在 [ **訂閱** ] 頁面中，尋找訂用帳戶 **轉移**。

    :::image type="content" source="images/direct/direct11.png" alt-text="查看轉移要求":::

6. 在 [訂用帳戶 **轉移**] 下，按一下要檢查的轉移要求。

7. 視需要按一下 [ **接受** ] (或 [ **拒絕** ]) 。

    :::image type="content" source="images/direct/direct12.png" alt-text="接受轉移要求":::

8. 等候傳送要求完成。

### <a name="indirect-provider---verify-transfer-request-is-complete"></a>間接提供者-驗證傳輸要求已完成

1. 成功完成傳送要求之後，請確認您可以看到訂用帳戶出現在 [ **訂閱**] 底下。

2. 通知轉換夥伴。

### <a name="transitioning-partner---verify-transfer-request-is-complete"></a>正在轉換夥伴-確認傳輸要求已完成

轉換夥伴應該執行下列作業：

1. 以系統 **管理員代理程式** 或 **銷售代理人**身分登入合作夥伴中心。

2. 在 [ **客戶** ] 頁面中，選取想要的客戶，然後按一下 [ **快速連結** ] 圖示，展開客戶的摘要視圖。

3. 按一下 [ **View 訂閱**]。

4. 在 [ **訂閱** ] 頁面中，尋找訂用帳戶 **轉移**。

5. 確認轉移要求已標示為 **完成**。

6. 確認訂用帳戶 () 在 [ **訂閱** ] 頁面中不會再顯示為 [作用中]：

   1. 如果這是 Azure 訂用帳戶 (MS-AZR-0003P Ms-azr-0145p) ，將不再列出該訂用帳戶。

   2. 如果這是以授權為基礎的訂用帳戶 (Office 365、Dynamics、Intune) ，其狀態會是 [已 **暫停**]。

   :::image type="content" source="images/direct/direct13.png" alt-text="訂用帳戶已暫止":::

### <a name="considerations"></a>考量

- **轉移之後，訂用帳戶識別碼將會不同。** 如果 Azure 訂用帳戶是 Azure 訂用帳戶 (MS-AZR-0003P Ms-azr-0145p) ，則會有 Azure 訂用帳戶識別碼，此識別碼會從先前的擁有者保留，並會出現在 Azure 入口網站中。

- **多個傳送要求無法參考相同的訂用帳戶。** 建立包含現有訂用帳戶的傳送要求之後，您就無法建立額外的轉移要求（包括相同的訂用帳戶），直到第一個轉移要求取消為止。

- **以授權為基礎的訂用帳戶附加元件必須連同其基礎訂用帳戶一起轉移。** 建立轉移要求時，如果您使用一或多個附加元件來挑選現有的訂用帳戶，則會自動將附加元件包含在傳送要求中。

- **訂用帳戶的授權計數變更將不會反映在現有的傳送要求中。** 建立包含現有訂用帳戶的傳送要求之後，您應該避免更新訂用帳戶的授權數量 (或相關聯的附加元件) 。 如果您這樣做，新數量不會反映在傳送要求中。 當間接提供者接受傳送要求之後，結果訂用帳戶將會有舊數量。 如果您想要將新數量傳送給間接提供者，您必須取消現有的轉移要求，並重新建立新的。

- **並非所有購買都可以使用自助訂用帳戶轉移來轉移。** 目前，您只能使用這項功能，將 O365 訂用帳戶和 Azure PAYG 訂用帳戶傳輸 (MS-AZR-0003P-Ms-azr-0145p) 。 不支援其他購買，包括 Azure 方案、Azure 保留實例、以詞彙為基礎的訂用帳戶，以及適用于 Azure Marketplace 的 SaaS 訂用帳戶。 您將會在 [提交轉移要求] 頁面中看到無法轉移訂用帳戶的原因。 若要轉移這些訂用帳戶，您必須 [取消現有的訂](create-a-new-subscription.md#suspend-or-cancel-a-subscription) 用帳戶，並透過間接提供者為客戶購買新的供應專案。

- **無法使用沙箱環境進行測試。**

## <a name="enroll-for-indirect-reseller-incentives"></a>註冊間接轉銷商獎勵

當您在現有的直接帳單合作夥伴租使用者上成功註冊為間接轉銷商之後，您將會收到邀請，在30天內註冊間接轉銷商獎勵。 此邀請是以目前與您的 CSP 合作夥伴租使用者相關聯的合作夥伴 MPN 帳戶為基礎。 邀請會傳送至與合作夥伴 MPN 帳戶相關聯的電子郵件地址。

您也有資格註冊相同合作夥伴租使用者的直接帳單獎勵方案。 您必須分別管理程式。
