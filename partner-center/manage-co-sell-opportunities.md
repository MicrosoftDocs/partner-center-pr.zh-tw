# <a name="manage-co-sell-opportunities-in-partner-center"></a>在合作夥伴中心中管理共同銷售商機

**適用於**

- 合作夥伴中心

**適當的角色**

- MPN 合作夥伴系統管理員
- 推薦管理員

利用共同銷售的機會。  共同銷售商機是多個合作物件的共同作業，可銷售符合客戶需求的產品。 本文將協助您瞭解如何在合作夥伴中心中使用共同銷售的強大功能，以有效地共同銷售和取得新客戶。

## <a name="types-of-co-sell-opportunities"></a>共同銷售商機的類型

**共同銷售** –這是一或多個 Microsoft 銷售代表積極參與交易，以協助解決客戶問題。 商機可能源自于 Microsoft 銷售代表邀請合作夥伴參與共同銷售活動的客戶帳戶，或是來自需要 Microsoft 銷售的額外協助來達成交易的合作夥伴。

**合作夥伴對合作夥伴 (P2P) ** –這是一個合作夥伴可以邀請 Microsoft 共同銷售生態系統中的其他合作夥伴進行共同作業。 合作夥伴可以選擇性地邀請 Microsoft 業務代表，以協助解決此交易。 這兩個夥伴和 Microsoft 都可以在建立交易之後獲得邀請，前提是在完成之前。  (注意，[完成] 包含 [成功]、[遺失]、[已拒絕] 和 [已) 過期

**私用交易** –這是當夥伴決定在合作夥伴中心中建立的交易上獨立運作時。 合作夥伴可以選擇在建立交易之後邀請 Microsoft 賣方，使其成為共同銷售活動。 這項交易的詳細資料並不會與 Microsoft 銷售團隊分享，直到合作夥伴明確尋求 Microsoft 銷售協助為止。

**解決方案評估 (SA) ** -這些類型的交易只能由解決方案評量商務小組通過的一組合作夥伴所建立，以便為使用或計畫使用 Microsoft 技術的客戶執行技術評量。

> [!NOTE]
> 私用交易（雖然它們不是真正的共同作業）包含在此，以提供簡單的途徑，讓您在任何非終端機階段中處理共同銷售商機的任何管線， (**成功、遺失、已拒絕和已過期** 的) 。

## <a name="navigating-the-co-sell-section"></a>導覽共同銷售區段

共同銷售區段中有三個最上層索引標籤。

**輸入** –在此索引標籤中提供從 Microsoft 銷售人員傳送給合作夥伴以進行共同銷售的所有商機。

**輸出** -此索引標籤中提供合作夥伴所建立的所有商機。合作夥伴可以建立三種不同的商機類別，即共同銷售、私用和合作夥伴。

我的最**愛–在**輸入或輸出共同銷售商機索引標籤中標示為 [我的最愛] 的商機將顯示在此索引標籤將任何商機標示為我的最愛，方法是按一下清單視圖中商機的心形圖示。 只要按一下相同的圖示，就可以從我的最愛移除商機。

## <a name="creating-a-co-sell-opportunity"></a>建立共同銷售商機

> [!IMPORTANT]
> 若要與 Microsoft 共同銷售，合作夥伴需要具備商業解決方案、參與的銷售小組，以及強大的上市策略。 合作夥伴也必須符合其他特定準則。 如需詳細資訊，請參閱 [成為共同銷售就緒](https://partner.microsoft.com/reach-customers/selling-with-microsoft#become-ready)。

您可以建立三種類型的商機。

### <a name="select-your-customer"></a>選取您的客戶

為此特定交易選取客戶。 輸入其位置，然後開始輸入名稱。 自動建議會根據搜尋準則開始顯示建議。 由於許多公司在相同位置有多個商務實體，您可能會看到多個結果。 如果您不確定要選擇哪一種，可以確認哪一個 D-N-S 號屬於您的客戶。 請務必為您正在使用的客戶選取完全相符的。

當您按一下 **[選取客戶**] 之後，系統會提示您輸入這項交易的主要連絡人的名稱、電話號碼和電子郵件地址。 這些詳細資料是選擇性的，只有當您打算邀請 Microsoft 銷售人員協助您參與參與時，才需要這些詳細資料。 選取 [下一步] 。

> [!IMPORTANT]
> 請確定您已取得客戶的同意，以提供其連絡人資訊給 Microsoft 進行交易。 Microsoft 可能會使用此資訊來直接與客戶聯繫。

### <a name="deal-details"></a>交易詳細資料

輸入目前交易的詳細資料。 當您主動與客戶密切合作來關閉交易時，定義如下的欄位可能會變更。

| **欄位名稱** | **強制/選用** | **詳細資料** |
|-------------|--------|-------|
|**交易名稱** | 強制性 | 好記的名稱，以便在稍後的時間點識別您的交易。 |
|**估計值** | 強制性 | 交易的價值取決於建立交易時所提供的資訊。|
|**預估結束日期**| 強制性| 您希望向客戶關閉交易的日期。 |
|**CRM 識別碼**| 選擇性 | 使用您的個別 CRM 中的商機識別碼來標記交易以供追蹤之用。|
|**行銷活動識別碼**| 選擇性 | 抓住產生交易的行銷活動。 如果您標記的所有交易都是來自具有相同識別碼的活動，此歸檔可協助您追蹤特定行銷活動的投資報酬率。|
|**備註**| 選擇性 | 更新所有的最新資訊，以提供您公司中其他員工處理相同交易或嘗試瞭解交易目前狀態的可見度。 您也可以使用此資訊作為記錄的通訊，以在 Microsoft 銷售人員/其他合作夥伴與您的公司之間進行討論。|

### <a name="add-team-members"></a>新增小組成員

新增交易詳細資料之後，請新增將處理此特定交易的員工。 您將需要輸入員工的名稱、電話號碼和電子郵件地址。 這些是必要的詳細資料，而且您必須至少有一個連絡人具有輸入的所有詳細資料，才能建立交易。 即使在建立交易之後，也可以變更這些詳細資料。 您先前交易最近的連絡人會顯示在右側，讓您快速將其新增到交易中。 針對 P2P 交易，小組可讓公司和公司的員工傳送邀請。

### <a name="add-solutions"></a>新增方案 (s) 

在本節中，您必須提供與此交易相關的解決方案的相關資訊。 這是必要的區段，您必須至少加入一個方案來建立交易。 您可以在建立交易之後變更解決方案詳細資料。 有多種類型的解決方案可以新增到交易中，如下所述：

- **我公司的解決方案：** 這些是貴公司所發行的共同銷售就緒解決方案
- **Microsoft：** 這些是 Microsoft 所擁有的解決方案
- **其他協力廠商解決方案：** 這些是由 Microsoft 共同銷售生態系統中其他合作夥伴發佈的共同銷售就緒解決方案
- **解決方案評定：** 這些是符合資格的合作夥伴可根據客戶需求選取的評量類型

> [!Important]
> 您只能為解決方案評估交易選取一種評量類型，而且不能新增任何解決方案。 選取解決方案評量之後，夥伴必須選擇要建立評量的位置。 這是正確獎勵支出的必要項。

提供解決方案資訊之後，請選取 [下一步] 移至您可以決定銷售類型的區段。 如果您選擇前三個選項的方案，而不是解決方案評估，您有三個選項：

**私用交易**：如果您未邀請 Microsoft 並在此步驟建立參與專案，它會屬於私用管線類型。 Microsoft 銷售人員將無法查看這項交易的詳細資料。

> [!Important]
> 交易註冊不適用於私用交易。 使用獎勵符合資格的解決方案來建立私用交易時，請務必小心，因為它們不符合合作夥伴中心的交易註冊資格。

**共同銷售交易：** 如果您選取的預設選項以外的任何選項「 **找出您想要的 microsoft 的協助類型**」，這項交易就會變成一項共同銷售交易，讓 Microsoft 賣方有可能協助您關閉交易。 Microsoft 的協助要求不保證 Microsoft 賣方會參與交易。 Microsoft 銷售代表有14天的時間可決定是否要參與。 在 [附注] 區段中，請務必找出您想要的説明類型。

**合作夥伴對合作夥伴 (P2P) 處理**：您可以按一下邀請夥伴連結，邀請其他夥伴進行交易。 以下是建立 P2P 交易的流程。

- **選取合作夥伴：** 按一下 [邀請夥伴] 之後，您就可以開始鍵入夥伴名稱，以取得符合您輸入之名稱的建議夥伴清單。 選取您感興趣的夥伴，以填入該夥伴的其他詳細資料。 您只能搜尋 Microsoft 共同銷售生態系統中的合作夥伴，並在合作夥伴中心中交易。

- **預估的結束日期：** 這是您預期受邀夥伴完成交易部分的日期。 日期已預先填入，因此您可以選擇只在必要時修改日期。 這是必要欄位，而且可以由您在建立交易後邀請的夥伴進行編輯。 您無法在建立交易之後修改此欄位。

- **估計值和貨幣：** 這是受邀夥伴將在整體交易中達成的交易價值。 請確定您在這裡輸入的值正確，讓受邀的合作夥伴可以決定是否要參與交易。 受邀的合作夥伴可以在建立交易之後變更此值。 您無法在建立交易之後修改此欄位。

- **附注：** 新增您邀請夥伴成為這項交易一部分的詳細資料。 詳細資訊將有助於受邀的夥伴決定是否要參與。

- **新增您的小組：** 將與受邀合作夥伴合作的員工加入公司。 如果受邀的夥伴接受交易，他們就可以加入自己的員工，讓兩家公司都能看到整個團隊的交易。 您只能在建立交易之前修改這些詳細資料。 在交易資料中輸入的員工詳細資料會預先填入，讓您更輕鬆地選擇要與此特定合作夥伴合作的員工。

- **新增解決方案：**  新增您希望受邀合作夥伴帶入資料表的解決方案。 至少有一個解決方案是強制性的。 受邀的合作夥伴可以在接受邀請之後修改解決方案。

- **識別說明的類型：** 識別說明的類型：最後，找出受邀夥伴需要的特定協助。

針對您想要邀請成為這項交易一部分的所有夥伴重複此步驟。 合作夥伴對合作夥伴的交易也可以讓 Microsoft 賣方參與 Microsoft 與合作夥伴的交易。 您也可以在建立交易之後，稍後邀請 Microsoft 和合作夥伴。

## <a name="responding-to-a-co-sell-opportunity"></a>回應共同銷售商機

每個商機會在其本身的生命週期內移動。

### <a name="received-stage"></a>已接收階段

在這個階段中，如果您收到來自 Microsoft 賣方或 Microsoft 共同銷售生態系統中其他合作夥伴的新共同銷售商機，請參閱詳細資料，如果您想要深入瞭解其商務需求，請洽詢客戶。 您可以在這個階段中採取兩個動作。 接受或拒絕參考：

- **接受：** 輸入交易的名稱、編輯預估的交易價值，以及根據您的審查預估的購買時間。 當您建立與客戶的連絡人之後，您應該在 [ **附注** ] 欄位中提供資訊，以深入瞭解客戶所要尋找的內容。 您可以選擇性地在此輸入您的 CRM 識別碼 (僅供參考) 、產生個別商機的行銷活動識別碼，並新增您公司中將會處理此交易的連絡人。

- 完成之後，選取 [下一步]****。 我們會將參考移至 **下一個階段**，這表示您打算主動與客戶合作來滿足其需求。 我們也會使用這項資訊來協助您在未來找出類似的交易。

- **拒絕**：選取您要拒絕交易的原因，並新增任何您想要包含的附注，然後選取 [ **關閉交易**]。 我們會將它封存為已 **拒絕** ，並通知 Microsoft 或傳送給您此機會的合作夥伴。

- 如果您未在分配的時間內回應 (目前14天) ，我們會將其封存為已 **過期** ，並通知 Microsoft 或傳送給您此機會的合作夥伴。

### <a name="accepted-stage"></a>接受的階段

努力與客戶完成交易。 如果您想要變更針對已接受的參考所提供的任何資訊，請選取 [ **編輯**]。 然後，您可以更新交易名稱、預估的購買日期、預估值、附注、CRM 識別碼和/或行銷活動識別碼。  您也可以選取 [ **新增您的小組** ]，以提供任何其他正在處理該交易之人員的名稱、電話號碼和電子郵件地址。 您也可以根據客戶需求來編輯方案。

依預設，您所建立的所有交易都是在接受的階段中。

開始處理交易之後，您可以在交易生命週期中標示銷售階段，以提供您所進行進度的詳細資料。 除了初始驗收或建立，以及最後的成功或遺失階段之外，還有四個階段（如下所述）。 提供這些詳細資料是選擇性的，但強烈建議您共用這些詳細資料，以在共同銷售交易中取得 Microsoft 銷售代表的適當協助。

:::image type="content" source="images/pscmigration/salesstage.png" alt-text="顯示銷售階段可標示之交易生命週期的影像。":::

> [!Note]
> 如果交易是解決方案評量的交易，銷售階段將會有所不同。 標示銷售階段也是解決方案評估交易的 **必要** 項。 只有在所有銷售階段都標示為已完成之後，才會啟用 [**贏**] 按鈕。

以下表格顯示銷售階段和相對於 Microsoft 合作夥伴中心推薦系統判斷的解決方案評量以外交易的相對百分比。

|**銷售階段名稱**|**銷售階段百分比**|**銷售階段的定義**|
|:----|:-----|:-----|
|建立時間|10%|建立輸出交易。|
|已接受|10%|接受輸入交易。|
|已合格|20%|先符合交易的價值和客戶需求，再繼續進行。|
|開發|40%|進一步開發以瞭解準備 POC 或正式提案所需的任何其他成品的詳細需求。|
|已提議|60%|根據客戶的需求對客戶提出正式提案。|
|已交涉|80%|根據提案來協調最終條款，以取得最終狀態-贏得或失去交易。|
|已拿下|100%|將交易標示為贏。|

當您完成時，您可以採取兩個動作中的其中一項，這會將交易標示為 **成功** 或 **遺失** 來回報結果。

> [!Note]
> 您的公司不一定要遵循相同的銷售階段。 這就是合作夥伴中心辨識交易銷售階段的方式，如果您使用 API 傳遞這些值，就會自動將您公司的階段對應到這些標準階段。 如果您使用合作夥伴中心 UX，則會使用資料表中所示的百分比來標記銷售階段。

> [!Important]
> 針對特定符合資格的解決方案，在您選取 [贏得] 之後，系統會要求您提供其他資訊來註冊您的交易。 Microsoft 將會審查您在此提供的資訊，並可能在審查期間要求提供其他詳細資料。 如需詳細資訊，請參閱[註冊您的交易](register-deals.md)。

只有在符合下列所有條件時，交易才有資格進行交易註冊。

1. Microsoft 受邀進行交易。
2. Microsoft 已接受邀請或將交易標示為贏。 您可以查看 Microsoft 在您的交易詳細資料下方的卡片，瞭解 Microsoft 的狀態。
3. 交易中有獎勵符合資格的解決方案。

> [!Important]
> 只有當您的公司名稱和獎勵符合資格的解決方案在與客戶的合約中清楚提及時，才註冊此交易。

如果交易有資格進行交易註冊，則會有額外的里程碑新增至稱為「交易註冊」的交易生命週期，如下所示。

:::image type="content" source="images/pscmigration/dealregstages.png" alt-text="顯示銷售階段可標示之交易生命週期的影像。":::

您可以選擇在將交易標示為獲勝之後，或稍後透過 [交易生命週期 **立即註冊** ] 按鈕，立即註冊交易。
一旦註冊交易，您就可以從相同的生命週期中查看交易驗證的進度。 如果您的公司需要採取任何動作，則交易生命週期視圖中會顯示適當的錯誤。 當交易驗證完成時，該交易就會進入「已關閉」狀態。

> [!Important]
> 交易評論和最終驗證狀態僅適用于 IP 共同銷售交易。

### <a name="combinations"></a>組合

下表顯示可在此交易階段受邀者的組合。

|**原始交易類型**|**誰可以受邀**|**備註**|
|-----|:-----|:-----|

### <a name="closed-stage"></a>關閉階段

這是所有商機的最終階段。 您可以在已關閉的階段中 **，查看贏、遺失、拒絕**和 **過期** 的所有交易。 在這個階段中，沒有任何可採取的動作。

## <a name="frequently-asked-questions"></a>常見問題集

**季.交易被標示為贏或遺失之後，是否可以進行編輯？**

否，交易在進入終端狀態後即無法修改。 過期、拒絕、贏和遺失為終端機狀態，無法進行進一步的更新。 當您將交易移至這些終端機狀態時，請務必小心。

**同比.我收到新的參考通知，但在合作夥伴中心中找不到它？**

如果您的公司有多個與相同 MPN 帳戶相關聯的租使用者，就會發生這種情況。 移至合作夥伴中心帳戶設定，並檢查與帳戶相關聯的租使用者。 然後建立要求連結租使用者的支援票證。 提供您在支援票證中登入的租使用者識別碼。

## <a name="getting-more-co-sell-opportunities"></a>取得更多共同銷售商機

以下是一些秘訣，可協助您獲得更多適用于您企業的共同銷售機會：

- **快速回應交易**。 當您及時回應連入要求時，我們會逐漸增加您在未來合作夥伴搜尋結果中的可見度。 請確定您的團隊快速回應並表明意向。
- **選擇您接受的交易**。 我們會監視您接受和拒絕的交易類型，並使用此資訊來協助您找出類似的交易。 接受不適合的交易並不會改善您的搜尋結果，而且可能會影響您收到的商機品質。
- **回報預估的交易大小、結束日期，以及交易的最後狀態**(已成交或未成交)。 我們將使用此資訊來繼續提供您品質的推薦。
