- **Azure 方案定價** 包含 azure 方案取用服務的定價資訊，以及 azure 方案保留定價。 價格為直接價格或 ERP，可針對任何指定的支援市場取得。 這些檔案中的資料會每天重新整理。
- **外幣** 會用來計算美元與夥伴當地貨幣之間的費用。 費率會在每個月的第一天更新，並在指定的月份取得費率。 這些檔案會在每個月的第一天更新。
- **Marketplace** 包含 Microsoft 商業市場 ISV 解決方案的定價。 每一市場都會抓取價格。 這些檔案中的資料會每天重新整理。

## <a name="price-changes"></a>價格變更

價格變更通常會發生。 合作夥伴可以查看價格清單預覽，以預期授權型供應專案的價格變更。 在合作夥伴中心儀表板上，開啟 [定價和供應專案] 頁面以查看價格清單預覽。

不過，以 Azure 使用量為基礎的定價沒有預覽。 合作夥伴可以透過使用 RateCard API 來趕上 Azure 耗用量價格的變更，這會傳回該日的計量定價。

|**產品類型**   |**產品範例**  |**預覽版可用** |**變更詳細資料**|
|-----------------------|:-----------------------|:-------------------|------------------|
|以授權為基礎|Office、Dynamics、Intune、Windows Enterprise|30天|列出預覽價格清單中標示為 CHNG 的價格變更|
|以使用量為基礎|Azure 資源|無法使用|在上個月的 [ **變更歷程記錄** ] 索引標籤中提供的變更記錄|
|軟體||無法使用|從 month 以手動方式比較價格清單與月份|
|Reservations|預先支付的虛擬機器|無法使用|從 month 以手動方式比較價格清單與月份|

以使用量為基礎的價格可能會在一個月內變更。 若要取得這些 Azure 資源的「目前」每日定價，合作夥伴必須呼叫 RateCard API。 

> [!Note] 
> 訂用帳戶價格變更只適用于續約。 合作夥伴的每月費用取決於購買的價格，或在建立訂用帳戶時的價格。 在取得年度期限之後，如果價格增加或減少，則在續約之前（通常是12個月的期限），合作夥伴不會收取變更價格的費用。

## <a name="add-on-offer-types"></a>附加元件供應專案類型

以授權為基礎的服務可作為基礎供應專案或附加元件取得。 只有基底供應專案可透過合作夥伴中心目錄進行探索及可購買。 購買基礎供應專案之後，必須套用附加元件。 以授權為基礎的價格清單 **次要授權類型** 資料行包含每個供應專案及其類型的相關資訊。 基礎供應專案在 [價格清單] 次要授權類型資料行中有 **非特定** 的值，而且可以在目錄中購買。 無法在目錄中購買 **附加** 元件的次要授權類型值。 若要購買這些附加元件：

1. 請參閱供應專案清單矩陣，查看需要購買的供應專案識別碼清單，然後才能購買附加元件。
2. 從目錄購買基礎供應專案
3. 從客戶清單中流覽至您的客戶。 按一下您剛剛購買之基本供應專案的訂用帳戶。 在 [管理訂用帳戶] 頁面上，您會看到可套用至基本供應專案的可用附加元件。

> [!Note] 
> 某些基底供應專案具有附加元件 **授權** 的 **單位類型** 值。 若為基本供應專案，這只表示您不需要在購買後指派使用者授權。 如果供應專案可以在類別目錄中購買，則不論使用者介面中的單位類型為何，都是 **基本供應** 專案。

## <a name="pricing-between-azure-and-non-azure"></a>Azure 與非 Azure 之間的定價

不同類型的供應專案價格各有不同。 以授權為基礎的定價通常是指定月份的每一授權數量。 以使用量為基礎的定價是藉由使用指定的資源（具有相關聯的計量識別碼）來決定。 取得 Azure 訂用帳戶不會收取合作夥伴的費用。 不過，合作夥伴會針對 Azure 訂用帳戶下不同部署所使用的資源付費。 以使用量為基礎的價格清單中的定價是以 Azure 中不同的資源計量識別碼來組織。

針對特定的資源類型（虛擬機器），Azure 保留是以詞彙為基礎的購買。 購買 Azure 保留可讓合作夥伴預先支付 (一年或三年期的條款) 並保留指定的虛擬機器。 保留可以節省合作夥伴金錢，並確保其虛擬機器永遠可在期限內使用。 合作夥伴可以根據以使用量為基礎的資源計量識別碼，來調整他們想要的保留。 計量識別碼在資源之間是一致的，不論夥伴是購買虛擬機器，或只是將虛擬機器部署為以使用量為基礎的資源。

## <a name="offer-limits"></a>供應專案限制

某些以授權為基礎的供應專案具有某些規則和限制，因此禁止相同客戶購買多個產品。 這些規則適用于大部分的試用版及許多小型企業優惠。 **小型企業** 供應專案是由具有最大授權計數小於300的供應專案所定義。

這些購買條件約束會定義為供應專案設定的一部分，而且可以藉由查看供應專案清單矩陣來找到。 兩個數據行會一起運作來定義強制執行：1。 供應專案限制範圍和2。 優惠限制。 在購買期間強制執行條件約束。 合作夥伴中心內的目錄，不允許合作夥伴購買比規則允許更多的供應專案。 任何違反條件約束的嘗試都會導致錯誤。

供應專案限制範圍會記錄為供應專案清單矩陣上的資料行，而且可以有 None、Lifetime 或並行的值。 

- 無條件約束可以購買 **無** 限制的優惠。
- **存留期** 優惠只能購買一次。
- **並行** 供應專案可依該供應專案的 **供應專案限制** 值所允許的次數多次購買。 大部分的試用版都有提供限制為 "1" 的存留期限制範圍。 大部分的小型企業供應專案都有一個並行供應專案限制範圍，其供應專案限制為 "2"。

> [!IMPORTANT]
> 即使已取消供應專案，仍會強制執行平行存取限制。 供應專案必須完全取消然後取消布建，才能釋出額外的空間，以允許另一次購買。
